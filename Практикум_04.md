### 4.2 Подсчёт консервативных позиций

Функция для определения доли совпадений в каждой колонке:

```python
def conservation_per_column(alignment):
    n = len(alignment)
    length = alignment.get_alignment_length()
    cons = []
    
    for i in range(length):
        col = alignment[:, i]
        
        # убираем гэпы для подсчёта частот
        col_nogap = col.replace('-', '')
        
        if len(col_nogap) == 0:
            cons.append(0)
        else:
            # доля наиболее частой буквы
            counts = {}
            for c in col_nogap:
                counts[c] = counts.get(c, 0) + 1
            
            max_count = max(counts.values())
            cons.append(max_count / len(col_nogap))
    
    return cons

cons = conservation_per_column(aln)

# Визуализация
import matplotlib.pyplot as plt

plt.plot(cons)
plt.xlabel("Позиция")
plt.ylabel("Доля совпадений")
plt.title("Консервативность по колонкам")
plt.show()
```

### 4.3 Поиск полностью консервативных позиций

```python
fully_conserved = [i for i, val in enumerate(cons) if val == 1.0]

```

## Часть 6: Построение филогенетического дерева

### 6.1 Расчёт попарных расстояний

Используем Biopython для расчёта матрицы расстояний:

```python
from Bio.Phylo.TreeConstruction import DistanceCalculator, DistanceTreeConstructor
from Bio import AlignIO
from Bio.Phylo import draw

aln = AlignIO.read("insulin_clustalo.aln", "clustal")

calculator = DistanceCalculator('identity')  # или 'blosum62' для белков
dm = calculator.get_distance(aln)

print(dm)
```

**Доступные матрицы расстояний:**
- `'identity'` — доля совпадений
- `'blosum62'` — матрица BLOSUM62 для белков
- `'transversion'` — для нуклеотидов

### 6.2 Построение дерева методом UPGMA

```python
constructor = DistanceTreeConstructor()
tree = constructor.upgma(dm)

# Визуализация
import matplotlib.pyplot as plt
from Bio import Phylo

Phylo.draw(tree)
```

#### Что такое UPGMA?

**UPGMA** (Unweighted Pair Group Method with Arithmetic Mean):
- Иерархический кластерный анализ
- Предполагает молекулярные часы
- Простой и быстрый метод
- Может давать неточные деревья

### 6.3 Сохранение дерева в Newick

```python
from Bio import Phylo

Phylo.write(tree, "insulin_tree.nwk", "newick")
```

**Формат Newick** — стандартный формат для хранения филогенетических деревьев

### 6.4 Построение дерева методом NJ (Neighbor-Joining)

```python
tree_nj = constructor.nj(dm)
Phylo.draw(tree_nj)
```

#### Что такое NJ (Neighbor-Joining)?

**NJ** (Neighbor-Joining):
- Не предполагает молекулярные часы
- Более точный метод
- Более вычислительно сложный
- Обычно предпочтительнее UPGMA

### 6.5 Визуализация в R с помощью ggtree

```r
library(ggtree)

ggtree(tree) +
```

## Часть 7: Продвинутые темы — профили и HMM

### 7.1 Создание профиля из выравнивания

**Профиль** — это матрица частот аминокислот в каждой позиции.

```python
def profile_from_alignment(alignment, alphabet='ACDEFGHIKLMNPQRSTVWY'):
    length = alignment.get_alignment_length()
    n_seq = len(alignment)
    profile = []
    
    for i in range(length):
        col = alignment[:, i].replace('-', '')
        
        if len(col) == 0:
            # Если только гэпы, частоты нулевые
            freq = [0.0] * len(alphabet)
        else:
            freq = [col.count(aa)/len(col) for aa in alphabet]
        
        profile.append(freq)
    
    return np.array(profile)

prof = profile_from_alignment(aln)
print(prof.shape)  # (длина выравнивания, 20)
```

**Структура профиля:**
- Строки = позиции выравнивания
- Столбцы = аминокислоты (или нуклеотиды)
- Значения = частоты встречаемости

### 7.2 Поиск по профилю (идея)

Профиль можно использовать для:
- **Поиска похожих последовательностей** в базе данных
- **Вычисления скора выравнивания** профиля с последовательностью
- Это основа для **PSI-BLAST**

## Часть 8: Применения — предсказание структуры

### 8.1 Коэволюция остатков

Множественное выравнивание позволяет выявить пары позиций, которые **изменяются согласованно (коэволюция)**.

**Почему это важно?**
- Такие пары часто находятся в **контакте в трёхмерной структуре**
- Помогает предсказать контакты в структуре

#### Упрощённый расчёт коэволюции

```python
from scipy.stats import chi2_contingency
import pandas as pd

def mutual_information(col1, col2):
    # col1, col2 – списки символов (без гэпов)
    data = []
    
    for a, b in zip(col1, col2):
        if a != '-' and b != '-':
            data.append((a, b))
    
    # Строим таблицу сопряжённости
    tbl = pd.crosstab(*zip(*data))
    chi2, p, dof, expected = chi2_contingency(tbl)
    
    # Информация может быть оценена через chi2
    return chi2 / (len(data) * len(tbl)**2)  # грубая оценка
```

## Контрольные вопросы (расширенные)

### Теоретические вопросы

1. **Почему точное (оптимальное) множественное выравнивание невозможно для большого числа последовательностей?**

   **Ответ:** Это NP-полная задача. При n последовательностях длины L число возможных выравниваний растёт экспоненциально. Уже для 10 последовательностей перебор всех вариантов становится невозможным.

2. **В чём заключается идея прогрессивного выравнивания? Какие у него недостатки?**

   **Ответ:** Идея: строим направляющее дерево на основе попарных выравниваний, затем выравниваем в порядке, определённом деревом. Недостаток: ошибки на ранних этапах накапливаются и не могут быть исправлены позже.

3. **Чем отличается алгоритм Clustal Omega от оригинального ClustalW?**

   **Ответ:** Clustal Omega использует HMM-профили и более сложную динамическую схему прогрессивного выравнивания. Это даёт лучшее качество, особенно для больших наборов.

4. **Как строятся профили и как их можно выравнивать между собой?**

   **Ответ:** Профиль — матрица частот. Выравнивание профилей: используется динамическое программирование, но вместо одного символа берём вектор частот.

5. **Что означают символы *, :, . в выравнивании, полученном в T-Coffee?**

   **Ответ:**
   - `*` = полная консервативность
   - `:` = сильная консервативность
   - `.` = слабая консервативность
   - пробел = неконсервативно

### Практические вопросы

6. **Как по множественному выравниванию построить филогенетическое дерево? В чём разница между UPGMA и Neighbor-Joining?**

   **Ответ:** Строим матрицу попарных расстояний, затем используем алгоритм кластеризации. UPGMA предполагает молекулярные часы (одинаковые скорости мутаций), NJ — нет. NJ обычно точнее.

7. **Для чего нужны логотипы последовательностей? Как интерпретировать высоту букв?**

   **Ответ:** Логотип показывает консервативность и предпочтение аминокислот в каждой позиции. Высота буквы пропорциональна её частоте, общая высота столбца отражает информационное содержание позиции.

8. **Как множественное выравнивание используется в предсказании структуры белков (например, в AlphaFold)?**

   **Ответ:** MSA содержит ковариационную информацию (коэволюция) — пары позиций, которые изменяются согласованно, часто находятся в контакте. AlphaFold2 использует эту информацию как мощный сигнал для предсказания структуры.

9. **В каких случаях стоит выбирать MAFFT, а в каких – T-Coffee?**

   **Ответ:** MAFFT — для больших наборов и когда нужна скорость. T-Coffee — когда критична точность и есть время (и ресурсы). Для среднего случая — Clustal Omega.

10. **Как можно оценить качество множественного выравнивания, если нет эталонного?**

    **Ответ:** Проверить консистентность с известными функциональными сайтами, структурой, проверить согласованность разных методов. Использовать меры вроде суммы парных совпадений (SPS).
