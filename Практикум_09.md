## Часть 1: Основные протоколы секвенирования (обзор)

### 1.1 Что можно делать с короткими ридами?

#### Анализ геномов

- **De novo сборка** — геном без референса
- **Ресеквенирование** — выравнивание на референс, поиск вариантов (SNP, инделы)

#### RNA-seq (транскриптомы)

- Измерение экспрессии генов
- Поиск новых изоформ
- Дифференциальная экспрессия

#### Метагеномика

- Полная сборка (shotgun metagenomics)
- Профилирование 16S рРНК

#### Хроматин

- **ChIP-seq** — места связывания белков
- **ATAC-seq** — открытый хроматин
- **Methyl-seq** — метилирование ДНК

#### Иммунные рецепторы

- Анализ B-клеток и T-клеток (BCR/TCR)

#### Single-cell RNA-seq

- Профиль отдельных клеток

### 1.2 Приготовление библиотек для разных протоколов

#### WGS (Whole Genome Sequencing)

```
ДНК → Фрагментация → Size selection → Адаптеры → ПЦР
```

#### RNA-seq

```
РНК → мРНК обогащение → Фрагментация → кДНК синтез → Адаптеры → ПЦР
```

#### Methyl-seq (Bisulfite-seq)

- Бисульфит натрия: неметилированный C → U → T
- Метилированный C остаётся C
- Сравнение с необработанной выявляет метилирование

#### ChIP-seq

```
Сшивка → Фрагментация → Иммунопреципитация → Очистка → Адаптеры → ПЦР
```

#### ATAC-seq

- Гиперчувствительная транспозаза Tn5
- Встраивает адаптеры в открытый хроматин
- Быстро, не требует фиксации

#### RepSeq (Immune Repertoire)

- Праймеры на константные и вариабельные сегменты TCR/BCR
- Амплификация только VDJ-области
- Парные риды для полного покрытия

#### Single-cell RNA-seq (10x Genomics)

- Эмульсия: одна клетка → гелевый шарик с баркодами
- **Cell barcode** — идентификатор клетки
- **UMI** (Unique Molecular Identifier) — удаление ПЦР-дубликатов
- Обратная транскрипция в капле

### 1.3 Длинные риды — для чего?

- Сборка сложных геномов (повторы, центромеры, теломеры)
- Полнотранскриптомный анализ (полные изоформы)
- Структурные варианты
- **Пример:** T2T consortium (2023) — полная сборка человеческого генома с центромерами

## Часть 2: Базы данных — где искать сырые риды?

### 2.1 Основные репозитории

| Репозиторий | URL | Регион |
|-----------|-----|--------|
| **NCBI SRA** | https://www.ncbi.nlm.nih.gov/sra | США |
| **EMBL-EBI ENA** | https://www.ebi.ac.uk/ena | Европа |
| **DDBJ DRA** | https://www.ddbj.nig.ac.jp/dra | Япония |

⚠️ **Все три синхронизируются** (International Nucleotide Sequence Database Collaboration)

### 2.2 GEO (Gene Expression Omnibus)

- Функциональные геномные данные (экспрессия, ChIP-seq, метилирование)
- Обработанные данные + ссылки на сырые риды в SRA
- Пример: GSE134296

### 2.3 Структура идентификаторов (префиксы NCBI)

| Префикс | Значение | Пример |
|---------|---------|--------|
| PRJNA/B | BioProject | PRJNA554659 |
| SRP | Study/Исследование | SRP214663 |
| SRX | Эксперимент | SRX6445021 |
| SAMN | Образец | SAMN12276403 |
| SRR | Запуск секвенирования | SRR9685614 |

### 2.4 Как найти данные по статье?

1. **Data Availability Statement** в статье
2. GEO accession (GSE...) или SRA BioProject (PRJNA...)
3. Таблица с образцами
4. Копируем SRR-номера для скачивания

### 2.5 SRA Run Selector

- https://www.ncbi.nlm.nih.gov/Traces/study/
- Вставляем SRP-номер
- Таблица со всеми образцами, метаданными, размерами
- Скачиваем Accession List (текстовый файл SRR)

## Часть 3: Скачивание данных — SRA Toolkit

### 3.1 Установка SRA Toolkit

```bash
# Через conda
conda install -c bioconda sra-tools

# Или вручную
wget https://ftp-trace.ncbi.nlm.nih.gov/sra/sdk/3.2.1/sratoolkit.3.2.1-ubuntu64.tar.gz
tar -xvf sratoolkit.3.2.1-ubuntu64.tar.gz
export PATH=$PATH:$(pwd)/sratoolkit.3.2.1-ubuntu64/bin
```

### 3.2 Основные команды

#### prefetch — быстрое скачивание в .sra

```bash
prefetch SRR9685614
```

Скачивает в `~/ncbi/public/sra/` или в текущую папку с `-O`

#### fasterq-dump — конвертация .sra в FASTQ

```bash
fasterq-dump SRR9685614/
```

Результат: `SRR9685614_1.fastq` и `SRR9685614_2.fastq` (парные риды)

⚠️ **Файлы не сжаты!** Дожимаем:

```bash
gzip SRR9685614_*.fastq
```

#### fastq-dump — прямое скачивание FASTQ

```bash
fastq-dump --split-files --gzip SRR9685614
```

- `--split-files` — разделить на R1 и R2
- `--gzip` — сразу сжать

⚠️ **Медленнее**, чем prefetch + fasterq-dump

### 3.3 Массовое скачивание по списку

```bash
for acc in $(cat SRR_Acc_List.txt); do
    prefetch $acc
    fasterq-dump $acc/
    gzip ${acc}_*.fastq
done
```

### 3.4 SRA-Explorer (альтернативный способ)

- https://ewels.github.io/sra-explorer/
- Вводим SRP/PRJNA номер
- Добавляем образцы в корзину
- Получаем готовый скрипт (прямые ссылки на FASTQ.gz)
- **Преимущество:** сразу сжатые файлы, wget/curl

## Часть 4: Базовая работа с FASTQ в командной строке

### 4.1 Полезные однострочники

#### Посчитать количество ридов

```bash
# Способ 1: через строки (разделить на 4)
echo $(zcat file_R1.fastq.gz | wc -l)/4 | bc

# Способ 2: через awk (каждая 4-я строка)
zcat file_R1.fastq.gz | awk 'NR%4==2' | wc -l

# Способ 3: через seqkit (если установлено)
seqkit stats file_R1.fastq.gz
```

#### Тестовый файл (первые N ридов)

```bash
# Первые 1 000 000 ридов (4 млн строк)
zcat file_R1.fastq.gz | head -4000000 | gzip > test_1M_R1.fastq.gz
```

#### Проверить длину ридов

```bash
zcat file_R1.fastq.gz | awk 'NR%4==2 {print length}' | sort | uniq -c
```

#### Извлечь заголовки

```bash
zcat file_R1.fastq.gz | awk 'NR%4==1' | head
```

### 4.2 Напоминание по командам

| Команда | Назначение |
|---------|-----------|
| `sort` | Сортировка строк |
| `uniq -c` | Подсчёт уникальных (только на сортированных) |
| `awk 'NR%4==2'` | Каждая 4-я строка, начиная со 2-й (сами последовательности) |
| `awk '{print substr($0, 1, 10)}'` | Первые 10 символов |

## Часть 5: Контроль качества — FastQC и MultiQC

### 5.1 Установка

```bash
conda install -c bioconda fastqc multiqc
```

### 5.2 Запуск на одном файле

```bash
fastqc SRR9685614_1.fastq.gz -o fastqc_results/
```

Результат: HTML-отчёт и ZIP-архив

### 5.3 Запуск на всех файлах

```bash
cd /path/to/fastqs
fastqc *.fastq.gz -o fastqc_results/

# Или параллельно (4 файла одновременно)
ls *.fastq.gz | parallel -j4 "fastqc {} -o fastqc_results/"
```

### 5.4 Интерпретация отчёта FastQC

#### Basic Statistics

Имя, кодировка (Sanger/Illumina 1.9), количество ридов, длина, % GC

#### Per base sequence quality ⭐ **САМЫЙ ВАЖНЫЙ**

- Зелёная зона — хорошо (Q > 28)
- Оранжевая — приемлемо (Q 20-28)
- Красная — плохо (Q < 20)

**Почему качество падает к концу?**
- Истощение реактивов
- Рассинхронизация фаз в кластерах (Illumina)
- R2 обычно хуже R1 (читается позже)

#### Per sequence quality scores

Распределение среднего качества по ридам. Пик влево (< 20) — много плохих ридов.

#### Per base sequence content

Доля A, C, G, T в каждой позиции. Должна быть ровной (если не обогащённый протокол).

Перекосы в начале → остатки адаптеров/праймеров.

#### Per sequence GC content

Распределение GC по ридам. Должно быть нормальным с пиком = ожидаемый GC организма.

**Бимодальное распределение → контаминация** (смесь организмов)

#### Per base N content

Доля неопределённых оснований (N). Много N → проблемы детекции.

#### Sequence duplication levels

Уровень дупликации ридов. Высокий (> 50%) может быть:
- ПЦР-дубликаты (мало материала)
- Истинные дубликаты (рРНК)
- Низкая сложность библиотеки

#### Overrepresented sequences

Слишком частые последовательности (обычно адаптеры или рРНК).

#### Adapter Content

% ридов с адаптерами в каждой позиции. Нужен тримминг.

### 5.5 MultiQC — объединение отчётов

```bash
cd /path/to/fastqc_results
multiqc .
```

Создаёт `multiqc_report.html` со сводными графиками по всем образцам.

## Часть 6: Система очередей SLURM на сервере

### 6.1 Зачем нужна?

- На сервере много пользователей, ресурсы ограничены
- Фронтенд доступен только 2 ядра и ограниченная память
- Для тяжёлых задач нужно запрашивать больше через SLURM

### 6.2 Структура SLURM-скрипта

```bash
#!/bin/bash

# Название задачи
#SBATCH --job-name=MyJob

# Количество ядер
#SBATCH --cpus-per-task=4

# Память (G, M суффиксы)
#SBATCH --mem=16gb

# Максимальное время (часы:минуты:секунды)
#SBATCH --time=02:00:00

# Лог-файл (стандартный вывод и ошибки)
#SBATCH --output=%j.%x.log

# Раздел (partition)
#SBATCH --partition=short

# Тип узла
#SBATCH --constraint=compute

# КОМАНДА
pwd
hostname
date
echo "Запуск задачи..."
fastqc myfile.fastq.gz -o fastqc_out/
```

### 6.3 Важные параметры

#### --partition (раздел/приоритет)

| Раздел | Время | Приоритет |
|--------|-------|----------|
| short | до 2 часов | Высокий |
| medium | до 16 часов | Средний |
| long | до 6 суток | Низкий |
| infinite | до 30 суток | Очень низкий |
| hpc | до 16 часов | Для ~112 ядер |
| gpu | до 16 часов | Видеокарты |

⚠️ **Если --time=03:00:00 и --partition=short, задача никогда не запустится!**

#### --cpus-per-task и --mem

- Не запрашивайте больше нужного — задача будет ждать
- Не запрашивайте меньше — может упасть с Out-Of-Memory

#### --output

- `%j` — номер задачи (job ID)
- `%x` — название задачи
- `%a` — номер в array (для параллельных)

### 6.4 Отправка задачи

```bash
sbatch slurm_template.sh
```

Вывод: `Submitted batch job 123456`

### 6.5 Мониторинг задач

#### Просмотр очереди

```bash
squeue
```

Выводит: JOBID, PARTITION, NAME, USER, ST (статус), TIME, NODES

- ST = R — Running (выполняется)
- ST = PD — Pending (ждёт)

#### Alias для своих задач

Добавьте в ~/.bashrc:

```bash
alias myq='squeue -u $USER -o "%.18i %.9P %.8j %.8u %.2t %.10M %.6D %R"'
source ~/.bashrc
```

#### Информация о запущенной задаче

```bash
sstat -j 123456 --format=JobID,MaxRSS,MaxVMSize,AveCPU
```

#### Информация о завершённой задаче

```bash
sacct -j 123456 --format=JobID,JobName,Partition,MaxRSS,MaxVMSize,Elapsed,State
```

- MaxRSS — реально использованная память (для оптимизации)

#### Отмена задачи

```bash
scancel 123456
```

### 6.6 Интерактивная сессия (для тестов)

```bash
srun --partition=short --cpus-per-task=4 --mem=8gb --time=01:00:00 --pty bash
```

Запрашивает ресурсы и даёт интерактивную оболочку на вычислительном узле.

### 6.7 Параллельный запуск с --array

Скрипт `run_fastqc_array.slurm`:

```bash
#!/bin/bash
#SBATCH --job-name=fastqc_array
#SBATCH --cpus-per-task=1
#SBATCH --mem=2gb
#SBATCH --time=01:00:00
#SBATCH --partition=short
#SBATCH --output=%j_%x_%a.log
#SBATCH --array=1-20   # 20 параллельных задач

FILES=($(ls /path/to/fastqs/*.fastq.gz))
FILE=${FILES[$SLURM_ARRAY_TASK_ID-1]}

fastqc $FILE -o /path/to/fastqc_results/
```

Отправка:

```bash
sbatch run_fastqc_array.slurm
```

SLURM запустит 20 отдельных задач с собственными логами.

## Контрольные вопросы

**1. Какие основные типы экспериментов с короткими ридами?**

**Ответ:** WGS, RNA-seq, метагеномика (16S и shotgun), ChIP-seq, ATAC-seq, Methyl-seq, RepSeq, scRNA-seq.

**2. Чем RNA-seq отличается от WGS?**

**Ответ:** WGS секвенирует всю геномную ДНК, RNA-seq только мРНК (обогащение poly-A). RNA-seq требует синтеза кДНК.

**3. Как работает бисульфитное секвенирование (Methyl-seq)?**

**Ответ:** Бисульфит натрия превращает неметилированные C в U (→T при ПЦР), метилированные C остаются. Сравнение выявляет метилирование.

**4. Что такое UMI и зачем в single-cell RNA-seq?**

**Ответ:** UMI (Unique Molecular Identifier) — случайная последовательность на каждой молекуле РНК. Позволяет удалить ПЦР-дубликаты и учесть истинное количество молекул.

**5. Три основных репозитория сырых данных?**

**Ответ:** NCBI SRA (США), EMBL-EBI ENA (Европа), DDBJ DRA (Япония). Синхронизируются.

**6. Что означают префиксы SRR, SRX, SRP, PRJNA?**

**Ответ:** SRR — запуск, SRX — эксперимент, SRP — исследование, PRJNA — BioProject.

**7. Разница prefetch и fasterq-dump?**

**Ответ:** prefetch скачивает в компактный формат .sra, fasterq-dump конвертирует в FASTQ. Вместе быстрее, чем fastq-dump.

**8. Как посчитать ридов в FASTQ?**

**Ответ:** `echo $(zcat file.fastq.gz | wc -l)/4 | bc` или `zcat file.fastq.gz | awk 'NR%4==2' | wc -l`

**9. Основные графики FastQC?**

**Ответ:** Per base quality (падение к концу), Per sequence GC (распределение), Adapter Content, Duplication levels, N content.

**10. Почему качество падает к концу на Illumina?**

**Ответ:** Истощение реактивов, рассинхронизация фаз в кластерах, фотоблидинг флуорофоров.

**11. Что означает бимодальное распределение GC?**

**Ответ:** Контаминация — смесь двух организмов с разным GC-составом.

**12. Зачем SLURM? Обязательные параметры?**

**Ответ:** Управление ресурсами на сервере. Обязательно: --job-name, --time, --partition, --mem (или --cpus-per-task), --output.

**13. Как посмотреть задачи и статус?**

**Ответ:** `squeue` — все, `squeue -u $USER` — свои, `myq` — alias для красивого вывода.

**14. Разница short, medium, long разделов?**

**Ответ:** short ≤2ч (высокий приоритет), medium ≤16ч (средний), long ≤6дней (низкий).

**15. Что если --partition=short и --time=03:00:00?**

**Ответ:** Задача никогда не запустится (3 часа превышают 2-часовой лимит short). Нужно увеличить time или выбрать другой partition.
