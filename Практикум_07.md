## Часть 1: Теоретические основы — от морфологии к молекулам

### 1.1 Зачем нужна филогенетика?

**Филогенетика** — наука об эволюционных взаимоотношениях между организмами или генами.

**Основные задачи:**

- Классификация организмов — построение "дерева жизни"
- Датировка эволюционных событий — время расхождения видов
- Изучение коэволюции — вирусы и хозяева
- Дизайн вакцин — выбор штаммов для вакцины против гриппа
- Криминалистика — доказательство заражения ВИЧ

### 1.2 Проблемы морфологической систематики

**Ограничения:**

- Нет целого организма (ископаемые фрагменты)
- Сложно отличить особи внутри вида
- Высокоуровневые сравнения невозможны (грибы vs животные)
- Микроорганизмы слишком однообразны
- Сложный жизненный цикл (полипы и медузы)
- **Криптические виды** — морфологически неотличимы, генетически разные
- **Аналогия vs гомология** — крылья насекомых и птиц (разное происхождение)

### 1.3 Молекулярная филогенетика

**Преимущества молекулярных данных:**

- Последовательности ДНК и белков
- Расположение генов в хромосомах
- Длина микросателлитов (внутривидовые сравнения)

### 1.4 Эволюционные события

| Событие | Описание |
|---------|---------|
| Замены (substitutions) | Точечные мутации |
| Инсерции/делеции (indels) | Вставки и удаления |
| Дупликации | Копирование генов |
| Транслокации | Перемещение фрагментов |
| Инверсии | Переворот участков |

### 1.5 Типы гомологии (КРИТИЧЕСКИ ВАЖНО!)

| Тип | Происхождение | Применение |
|-----|--------------|-----------|
| **Ортологи** | Видообразование | Деревья видов ✓ |
| **Паралоги** | Дупликация гена | Деревья генов |
| **Ксенологи** | Горизонтальный перенос | В основном бактерии |
| **Гомеологи** | Гибридизация | Редко |
| **Онологи** | Дупликация генома | Редко |

⚠️ **Важно:** Для дерева видов используйте ортологи. Паралоги отражают историю дупликаций.

## Часть 2: Строение филогенетических деревьев

### 2.1 Основные элементы

| Элемент | Описание |
|---------|---------|
| **Узлы (nodes)** | Таксоны или предки |
| **Ветви (branches)** | Эволюционные линии |
| **Корень (root)** | Общий предок |
| **Клада (clade)** | Монофилетическая группа |

### 2.2 Информативные свойства

- **Топология** — структура ветвления (родство)
- **Длина ветвей** — количество замен или время
- **Наличие корня** — направление эволюции

### 2.3 Неинформативные свойства

- Форма, ориентация, углы — просто для наглядности

### 2.4 Типы групп

| Тип группы | Описание | Пример |
|-----------|---------|--------|
| **Монофилетическая** | Все потомки предка | Млекопитающие |
| **Парафилетическая** | Только часть потомков | Рептилии без птиц |
| **Полифилетическая** | От разных предков | Теплокровные животные |

### 2.5 Пример: рептилии как парафилетическая группа

Птицы произошли от динозавров. **Рептилии без птиц** = парафилетическая группа.

### 2.6 Типы деревьев по длине ветвей

| Тип | Описание |
|-----|---------|
| **Кладограмма** | Только топология |
| **Филограмма** | Длина ~ количество замен |
| **Хронограмма** | Длина ~ время |

### 2.7 Укоренённые и неукоренённые

- **Неукоренённое** — только родство, без направления
- **Укоренённое** — показывает корень и направление эволюции

## Часть 3: Выбор последовательностей для филогенетики

### 3.1 Требования к маркерам

**Универсальность** — есть у всех организмов
**Копийность** — легко секвенируются
**Разнообразие оптимальное** — не слишком мало, не слишком много

### 3.2 Классические маркеры

| Маркер | Организмы | Использование |
|--------|-----------|----------------|
| **16S рРНК** | Прокариоты | Бактерии, археи |
| **18S рРНК** | Эукариоты | Грибы, простейшие |
| **mtДНК** | Животные | Близкие виды |
| **cpДНК** | Растения | Растения |
| **Консервативные гены** | Все | Глубокие отношения |

### 3.3 Внешняя группа (outgroup)

- Заведомо не принадлежит к исследуемой группе
- Должна быть достаточно близка для выравнивания
- Необходима для укоренения дерева

### 3.4 Несколько генов

| Подход | Описание | Использование |
|--------|---------|----------------|
| **Суперматрица** | Все гены в одно выравнивание | Часто ✓ |
| **Супердерево** | Дерево для каждого гена | Реже |

## Часть 5: Модели эволюции нуклеотидных последовательностей

### 5.1 P-distance (простейшее)

```
p = Nd / N
где Nd — несовпадающие позиции, N — общая длина
```

**Проблема:** не учитывает множественные замены

### 5.2 Модель Джукса–Кантора (JC69, 1969)

- Равные частоты нуклеотидов (πA = πC = πG = πT = 0.25)
- Все замены равновероятны

```
d_JC = -3/4 × ln(1 - 4/3 × p)
```

### 5.3 Модель Кимуры (K80/K2P, 1980)

- Различает транзиции (Ts) и трансверсии (Tv)
- Равные частоты нуклеотидов
- κ = α/β (скорости)

### 5.4 Более сложные модели

| Модель | Особенности |
|--------|-----------|
| **F81** | Разные частоты, равные скорости |
| **HKY85** | Разные частоты, разные Ts/Tv |
| **TN93** | Разные частоты, разные Ts и Tv |
| **GTR** | Самая общая (6 скоростей + 4 частоты) |

### 5.5 Выбор модели

В MEGA, jModelTest, ModelFinder — автоматический выбор по AIC, BIC.

### 5.6 Практическое задание

В MEGA:
```
Distance → Compute Pairwise Distances
Выберите модель: Jukes-Cantor
Посмотрите матрицу расстояний
Повторите с Kimura 2-parameter
```

## Часть 6: Методы построения деревьев

### 6.1 Общая схема

1. Выбор последовательностей (с внешней группой)
2. Множественное выравнивание
3. Фильтрация (тримминг)
4. Выбор модели эволюции
5. Построение дерева
6. Проверка (бутстрап)
7. Укоренение
8. Датировка (если есть калибровки)
9. Визуализация

### 6.2 Методы на основе расстояний

#### UPGMA (Unweighted Pair Group Method with Arithmetic Mean)

**Алгоритм:**
1. Найти две ближайшие последовательности
2. Объединить в кластер
3. Пересчитать расстояния (среднее)
4. Повторять

**Предположение:** молекулярные часы (проблема!)

#### Neighbor-Joining (NJ)

**Алгоритм:**
1. Начать со звездообразного дерева
2. Исправленная матрица: D*ᵢⱼ = (n-2)Dᵢⱼ - TotalDist(i) - TotalDist(j)
3. Найти пару с минимальным D*
4. Объединить и пересчитать

**Плюсы:** не требует молекулярных часов, быстрее ML

### 6.3 Дискретные методы (используют сами последовательности)

#### Максимальная парсимония (MP)

- Выбирается дерево с наименьшим числом замен
- Хорошо для близких последовательностей
- Страдает от "long branch attraction"

#### Максимальное правдоподобие (ML)

- Максимизирует вероятность данных
- Учитывает модель эволюции
- Точнее, но медленнее
- Программы: RAxML, IQ-TREE, PhyML

#### Байесовский подход (BI)

- Использует MCMC
- Дает апостериорные вероятности узлов
- Программы: MrBayes, BEAST

### 6.4 Сложность: сколько деревьев?

Число неукоренённых деревьев для n таксонов:

```
(2n-5)!! = 1×3×5×...×(2n-5)
```

Для 22 таксонов ~ 0.5 моля! Полный перебор невозможен.

## Часть 7: Оценка достоверности — Бутстрап (Bootstrap)

### 7.1 Идея

1. **Генерируем** псевдовыборки (100, 1000, 10000 раз)
   - Случайный выбор колонок выравнивания с возвращением
2. **Строим** дерево для каждой псевдовыборки
3. **Считаем** процент случаев, когда узел сохранился
4. **Наносим** значение на узлы финального дерева

### 7.2 Интерпретация

| Значение | Интерпретация |
|----------|----------------|
| > 90% | Очень надежен ✓ |
| 70-90% | Надежен |
| 50-70% | Неуверенный |
| < 50% | Не поддерживается |

При низких значениях — изобразите как политомию (неразрешённый узел)

### 7.3 Практическое задание в MEGA

```
Phylogeny → Construct/Test Neighbor-Joining Tree
Test of phylogeny: Bootstrap method
No. of Bootstrap replications: 100 (для публикации 1000)
Substitution model: Jukes-Cantor или Kimura 2-parameter
Gaps/Missing Data Treatment: Pairwise deletion
```

## Часть 8: Укоренение (Rooting) дерева

### 8.1 Зачем укоренять?

- Неукоренённое дерево показывает только родство
- Для направления эволюции нужен корень
- Для определения предковых состояний

### 8.2 Методы укоренения

| Метод | Описание | Надежность |
|-------|---------|-----------|
| **Внешняя группа** | Заведомо далёкий таксон | Лучший способ ✓ |
| **Серединная точка** | Середина длинного пути | Если нет outgroup |
| **Случайная последовательность** | Искусственно добавляют | Редко |

### 8.3 Практическое задание в MEGA

```
После построения дерева:
Root → By Outgroup
Выберите: Аллигатор как outgroup
```

## Часть 9: Визуализация деревьев

### 9.1 Формат Newick

Стандартный формат хранения деревьев:

```
((((68:4.0,69:2.0)67:1.0,66:5.0,70:2.0)65:1.0,(80:1.0,81:1.0)72:1.0)55:3.0,...);
```

Скобки — группы, двоеточия — длины ветвей

### 9.2 Визуализация в MEGA

- Встроенный Tree Explorer
- Менять стиль (прямоугольный, круговой, радиальный)
- Показывать бутстрап-значения
- Изменять цвета
- Экспортировать в SVG, PDF

### 9.3 Визуализация в R с ggtree

```r
# Установка
if (!require("BiocManager", quietly = TRUE))
    install.packages("BiocManager")
BiocManager::install("ggtree")

library(ggtree)
library(ape)

# Чтение дерева в Newick
tree <- read.tree("ef1a_tree.nwk")

# Простая визуализация
ggtree(tree) + 
  geom_tiplab() + 
  geom_nodelab(aes(label = bootstrap), hjust = 1.3) +
  xlim(0, 0.5) +
  theme_tree2()

# Радиальное дерево
ggtree(tree, layout = "circular") + 
  geom_tiplab(aes(angle = angle), size = 3)
```

## Часть 13: Продвинутые темы

### 13.1 Деревья генов и деревья видов

Из-за неполной сортировки линий, горизонтального переноса или дупликаций деревья генов могут не совпадать с деревом видов. **Решение:** несколько генов + методы коалесценции (ASTRAL)

### 13.2 Сети вместо деревьев

При гибридизации или горизонтальном переносе — используйте сети.
Программы: SplitsTree, PhyloNet

### 13.3 Филогеномика

Использование полных геномов или больших наборов генов. Требует мощных вычислительных ресурсов.

## Контрольные вопросы (расширенные)

**1. В чём разница между гомологией и аналогией? Приведите примеры.**

**Ответ:** Гомология — общее эволюционное происхождение (глобины у млекопитающих). Аналогия — сходная функция, разное происхождение (крылья насекомых и птиц).

**2. Чем ортологи отличаются от паралогов? Почему для дерева видов нужны ортологи?**

**Ответ:** Ортологи возникли при видообразовании (отражают историю видов). Паралоги — при дупликации (отражают историю генов). Для дерева видов нужны ортологи.

**3. Какие проблемы морфологической систематики решает молекулярная филогенетика?**

**Ответ:** Ископаемые фрагменты, криптические виды, микроорганизмы, высокоуровневые сравнения, сложные жизненные циклы.

**4. Что такое криптические виды и как их обнаружить?**

**Ответ:** Морфологически неотличимые, но генетически разные виды. Обнаружить: анализ ДНК, филогенетические деревья.

**5. Почему 16S рРНК — хороший маркер для бактерий?**

**Ответ:** Универсален (у всех), высокая копийность, содержит консервативные и вариабельные участки, позволяет строить деревья от близких до далёких видов.

**6. Что такое внешняя группа и зачем она нужна?**

**Ответ:** Таксон, заведомо не принадлежащий к исследуемой группе. Нужна для укоренения дерева (корень между outgroup и группой).

**7. В чём разница между UPGMA и Neighbor-Joining?**

**Ответ:** UPGMA предполагает молекулярные часы (может дать неверную топологию), NJ — нет. NJ более robust, не требует постоянной скорости эволюции.

**8. Что такое бутстрап и как интерпретировать значения?**

**Ответ:** Псевдовыборки из выравнивания, дерево для каждой. Значение (%) — доля деревьев, где узел сохранился. > 70% — надежен, < 50% — не поддерживается.

**9. Какие параметры учитывают модели (JC69, K80, HKY85, GTR)?**

**Ответ:** JC69 — равные всё; K80 — Ts/Tv; HKY85 — Ts/Tv + частоты; GTR — все 6 скоростей + частоты.

**10. Разница между кладограммой, филограммой, хронограммой?**

**Ответ:** Кладограмма — только топология; филограмма — длина ~ замены; хронограмма — длина ~ время.

**11. Что такое политомия (неразрешённый узел)?**

**Ответ:** Узел с более чем двумя ветвями. Говорит, что данные не поддерживают однозначное разветвление.

**12. Почему деревья генов не совпадают с деревом видов?**

**Ответ:** Неполная сортировка линий, горизонтальный перенос, дупликации. Решение: несколько генов + коалесценция.

**13. Программы для байесовской филогенетики и датировки?**

**Ответ:** MrBayes (филогенетика), BEAST (датировка), r8s (непараметрическая датировка).

**14. Как визуализировать дерево в R? Какие пакеты?**

**Ответ:** Пакет ggtree (основной), ape (read/write), phangorn (анализ). ggtree позволяет цвета, метки, макеты.

**15. Суперматрица vs супердерево?**

**Ответ:** Суперматрица — все гены в одно выравнивание (часто ✓). Супердерево — дерево для каждого, потом комбинация (редко).
