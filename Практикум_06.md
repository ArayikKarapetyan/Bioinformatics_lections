## Часть 1: Теоретические основы

### 1.1 Марковские процессы

**Марковский процесс** — случайный процесс, где вероятность будущего состояния зависит только от текущего состояния (свойство отсутствия памяти).

#### Пример: погода

Состояния: Солнечно (S), Облачно (C), Дождливо (R)

Матрица переходов:
```
     S    C    R
S   0.8  0.15 0.05
C   0.3  0.5  0.2
R   0.2  0.3  0.5
```

### 1.2 Скрытые марковские модели (HMM)

В HMM состояния не наблюдаются напрямую, только эмиссии (наблюдаемые символы).

#### Пример: друг в городе

- **Скрытые состояния:** погода (S, C, R)
- **Наблюдаемые:** настроение друга (Счастлив, Грустен)

Матрица эмиссий:
```
         Счастлив  Грустен
Солнечно   0.8      0.2
Облачно    0.6      0.4
Дождливо   0.1      0.9
```

### 1.3 Компоненты HMM

| Компонент | Описание |
|-----------|---------|
| **Q** | Скрытые состояния {q₁, q₂, ..., qₖ} |
| **V** | Наблюдаемые символы {v₁, v₂, ..., vₘ} |
| **A** | Матрица переходов aᵢⱼ = P(qⱼ\|qᵢ) |
| **B** | Матрица эмиссий bⱼ(k) = P(vₖ\|qⱼ) |
| **π** | Начальное распределение πᵢ = P(qᵢ первым) |

Вероятность события:
```
P(x, путь) = π × ∏ переходы × ∏ эмиссии
```

### 1.4 Три основные задачи HMM

| Задача | Алгоритм | Применение |
|--------|----------|-----------|
| Оценка вероятности | Forward | Проверка принадлежности |
| Декодирование | **Viterbi** | **Выравнивание последовательности** |
| Обучение параметров | Baum-Welch | Построение модели |

**В HMMER используется Viterbi** для выравнивания на профиль.

## Часть 2: Профильные HMM для белков

### 2.1 Архитектура HMM (Plan7)

| Тип состояния | Описание |
|---------------|---------|
| **Match (M)** | Консервативные колонки в выравнивании |
| **Insert (I)** | Дополнительные аминокислоты |
| **Delete (D)** | Пропуск консервативной позиции |

Переходы: M→M, M→I, M→D, I→I, I→M, D→D, D→M

### 2.2 Построение профиля

**Шаги:**

1. Определить консервативные колонки в MSA
2. Подсчитать частоты эмиссий для match states
3. Подсчитать вероятности переходов
4. Добавить псевдоотсчёты для редких символов

### 2.3 Псевдоотсчёты

Предотвращают нулевые вероятности для редких символов. Часто смешивают:
- Фоновые частоты аминокислот
- Матрицы замен (BLOSUM)

## Часть 3: Pfam и InterPro

### 3.1 Pfam база данных

**Pfam** — база профильных HMM для белковых семейств.

Каждое семейство имеет:
- Accession (PF00021)
- Seed alignment (репрезентативные)
- Full alignment (все найденные)
- Profile HMM

### 3.2 InterPro

Объединяет: Pfam, SMART, PROSITE, PANTHER...

### 3.3 Типы записей

| Тип | Описание |
|-----|---------|
| Family | Общее происхождение |
| Domain | Структурная единица |
| Repeat | Повторы |
| Motif | Консервативный участок |
| Clans | Объединение по структуре |

## Часть 5: HMMER в командной строке

### 5.1 Установка

```bash
conda install -c bioconda hmmer
hmmsearch -h  # проверка
```

### 5.2 Подготовка данных

```bash
# Создаём FASTA с последовательностями
cat > sequences.fasta << EOF
>Seq1
ACDEFGH...
>Seq2
ACDEFGH...
EOF

# Выравниваем
clustalo -i sequences.fasta -o aln.fasta --outfmt=fasta

# Конвертируем в Stockholm
clustalo -i sequences.fasta -o aln.sto --outfmt=sto
```

### 5.3 Построение HMM (hmmbuild)

```bash
hmmbuild profile.hmm aln.sto
```

Результат: файл profile.hmm с моделью

### 5.4 Поиск по базе (hmmsearch)

```bash
hmmsearch --tblout results.txt profile.hmm database.fasta
```

Результаты в tabular формате:
- target name
- accession
- E-value
- bit score
- координаты выравнивания

### 5.5 Анализ в Python

```python
import pandas as pd

# Парсим результаты
data = []
with open("results.txt") as f:
    for line in f:
        if not line.startswith("#"):
            parts = line.split()
            if len(parts) >= 5:
                data.append({
                    "target": parts[0],
                    "evalue": float(parts[4]),
                    "score": float(parts[5])
                })

df = pd.DataFrame(data)
print(f"Найдено: {len(df[df['evalue'] < 1e-5])} значимых")
```

### 5.6 Локальный hmmscan

Если есть локальная копия Pfam:

```bash
hmmscan --tblout results.txt Pfam-A.hmm protein.fasta
```

## Часть 6: Сравнение HMMER и PSI-BLAST

### 6.1 Пример с глобинами

Запрос: 3 глобина + 1 миоглобин позвоночных
База: UniProt (~1060 глобинов)

| Метод | Найдено | Время |
|-------|---------|-------|
| PSI-BLAST | 915 | 9 сек |
| HMMER3 | 1002 | 8 сек |

**HMMER находит больше**, включая растительные и бактериальные

### 6.2 Почему HMMER чувствительнее?

- Полная вероятностная модель (не просто PSSM)
- Позиционно-специфичные вероятности вставок/делеций
- Лучше для низкой идентичности (<20%)

## Контрольные вопросы

**1. Чем HMM отличается от обычной марковской цепи?**

В обычной цепи состояния наблюдаемы; в HMM состояния скрыты, видны только эмиссии.

**2. Три основные задачи HMM и какая используется в HMMER?**

Forward (оценка), Viterbi (декодирование — **в HMMER**), Baum-Welch (обучение)

**3. Match, Insert, Delete states?**

Match — консервативные позиции; Insert — вставки; Delete — пропуски

**4. Как строить HMM по выравниванию?**

Определить консервативные колонки, подсчитать частоты, добавить псевдоотсчёты

**5. Seed vs full alignment в Pfam?**

Seed — малое курируемое; full — все найденные

**6. Что такое клан?**

Объединение семейств по сходству структуры, даже если низкая последовательностная гомология

**7. HMMER vs PSI-BLAST (пример с глобинами)?**

HMMER находит 1002, PSI-BLAST 915. HMMER точнее моделирует вставки/удаления

**8. Режимы HMMER?**

phmmer (белок vs белки), hmmscan (белок vs HMM), hmmsearch (HMM vs белки), jackhmmer (итеративно)

**9. Формат Stockholm?**

Формат для выравниваний с метаинформацией, используется в HMMER

**10. E-value в HMMER vs BLAST?**

HMMER учитывает вероятностную модель более надежно, лучше справляется со структурой выравнивания

**11. CpG-островки с HMM?**

Двухсоставная модель, алгоритм Витерби для декодирования геномных последовательностей

**12. 3Dpatch?**

Наносит консервативность из HMM на 3D-структуру, помогает определить функциональные участки
