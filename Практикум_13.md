## Часть 1: Введение в транскриптомику

### 1.1 ДНК vs РНК — в чём разница?

| Характеристика | ДНК | РНК |
|---|---|---|
| **Копий на клетку** | Фиксировано (2) | Варьирует (0 — десятки тысяч) |
| **Стабильность** | Высокая | Низкая (быстро деградирует) |
| **Информативность** | Генетические варианты | Уровень экспрессии, изоформы |
| **Реплики** | Обычно не нужны | Обязательны (3+) |

### 1.2 Задачи транскриптомики

✅ **Профиль экспрессии** — какие гены активны в клетке/ткани
✅ **Дифференциальная экспрессия** — сравнение между условиями
✅ **Определение изоформ** — варианты сплайсинга
✅ **Обнаружение мутаций** — SNP, инделы в транскриптах
✅ **Коэкспрессия генов** — сети взаимодействий
✅ **Сборка de novo** — неизвестный геном
✅ **Single-cell RNA-seq** — уровень отдельных клеток

### 1.3 Типы РНК в клетке

| Тип РНК | % от всей РНК | Функция |
|---------|---|---------|
| **мРНК** | 1-5% | Кодирует белки (интересует исследователей) |
| **рРНК** | 80-90% | Рибосомы (мешает, удаляют) |
| **тРНК** | 5-15% | Трансляция |
| **мнРНК** | ~ | Регуляция (микро-РНК) |
| **другие** | ~ | lncRNA, snoRNA, snRNA |

### 1.4 От транскрипции к белку

```
ДНК (ген) 
    ↓ транскрипция
пре-мРНК (с интронами)
    ↓ сплайсинг
мРНК (экзоны)
    ↓ трансляция
белок
```

⚠️ **Альтернативный сплайсинг:** из одного гена → разные изоформы мРНК → разные белки

## Часть 2: Дизайн эксперимента RNA-seq

### 2.1 Факторы, влияющие на результат

**Биологические факторы:**
- Возраст, пол, генотип донора
- Условия культивирования клеток
- Время после смерти организма

**Технические факторы:**
- Метод стабилизации РНК
- Метод выделения РНК
- Обогащение (polyA vs RiboZero)
- Batch effect (разные дни, люди, партии реактивов)

**Деградация РНК:**
- RIN < 7 → низкое качество → искажённые результаты

### 2.2 Batch effect — как бороться?

✅ Рандомизировать образцы по дням/партиям
✅ Один лот реактивов для всего эксперимента
✅ Один человек готовит все библиотеки
✅ Включить batch в статистическую модель

### 2.3 Обогащение по мРНК vs удаление рРНК

| Метод | Протокол | Преимущества | Недостатки |
|---|---|---|---|
| **PolyA selection** | Магнитные шарики + oligo-dT | ~70% на экзоны | Не работает с деградированной РНК |
| **RiboZero** | Удаление рРНК зондами | Некодирующие РНК | Только ~40% на экзоны |

### 2.4 Сколько секвенировать?

- **Высокоэкспрессирующиеся гены:** 5-10 млн ридов/образец
- **Все гены включая низкие:** 20-50 млн ридов/образец
- **Реплики:** минимум 3 биологических реплики на условие
- **Золотое правило:** больше реплик > больше ридов (при фиксированном бюджете)

### 2.5 Направленные (stranded) библиотеки

- **Стандартные:** теряют информацию о цепи
- **Направленные:** сохраняют направление (dUTP в синтезе кДНК)
- **Преимущества:** антисмысловые транскрипты, лучше аннотация

## Часть 3: Контроль качества и тримминг

### 3.1 Общий пайплайн

```
FASTQ (сырые риды)
    ↓
FastQC (QC)
    ↓
Тримминг (fastp, Trimmomatic)
    ↓
FastQC после тримминга
    ↓
MultiQC (сводный отчёт)
    ↓
Выравнивание/сборка
```

### 3.2 Проверка на контаминацию (fastq_screen)

Сравнивает риды с геномами (человек, мышь, бактерии, фаги, адаптеры)

**Обнаруживает:**
- Контаминацию организмами
- Рибосомальную РНК (если не удалена)
- Адаптеры

### 3.3 GC-состав

- **Ожидаемое:** унимодальное распределение
- **Бимодальное:** контаминация или рРНК

### 3.4 UMI (Unique Molecular Identifiers)

Случайные баркоды в адаптерах → отличают молекулы мРНК от ПЦР-дубликатов

```bash
fastp -i R1.fastq.gz -I R2.fastq.gz \
    --umi --umi_loc=read1 --umi_len=10 ...
```

## Часть 4: Выравнивание RNA-seq на геном

### 4.1 Специфика выравнивания РНК

- Рид может пересекать границу экзон-экзон (сплайс-сайт)
- Алгоритм должен "разрезать" рид и выровнять части отдельно
- Требуется аннотация сплайс-сайтов (GTF/GFF)

### 4.2 Популярные сплайс-чувствительные выравниватели

| Программа | Скорость | Память | Особенности |
|-----------|----------|--------|-----------|
| **STAR** | Очень быстро | Много (30-50 ГБ) | Рекомендуется для больших проектов |
| **HISAT2** | Средняя | Средняя (~10 ГБ) | Иерархические индексы |
| **TopHat2** | Медленно | Эффективно | Исторический, редко используется |
| **Salmon/Kallisto** | Очень быстро | Мало | Псевдовыравнивание (не требует сплайсинга) |

### 4.3 STAR — Spliced Transcripts Alignment

#### Индексация (один раз)

```bash
STAR --runMode genomeGenerate \
     --runThreadN 16 \
     --genomeDir /path/to/STAR_index \
     --genomeFastaFiles reference.fa \
     --sjdbGTFfile annotation.gtf \
     --sjdbOverhang 100
```

- `--sjdbOverhang` = длина рида − 1

#### Выравнивание

```bash
STAR --runThreadN 16 \
     --genomeDir /path/to/STAR_index \
     --readFilesCommand zcat \
     --readFilesIn sample_R1.fastq.gz sample_R2.fastq.gz \
     --quantMode GeneCounts \
     --outFileNamePrefix sample_ \
     --outSAMtype BAM SortedByCoordinate
```

**Параметры:**
- `--quantMode GeneCounts` — подсчёт ридов на ген из GTF
- `--outSAMtype BAM SortedByCoordinate` — сортированный BAM

**Выходные файлы:**
- `sample_Aligned.sortedByCoord.out.bam` — выравнивания
- `sample_Log.final.out` — метрики (% уникально выровненных)
- `sample_ReadsPerGene.out.tab` — количество ридов на ген

### 4.4 HISAT2

#### Индексация

```bash
hisat2-build -p 16 reference.fa reference_index
```

#### Выравнивание

```bash
hisat2 -p 16 \
       -x reference_index \
       -1 sample_R1.fastq.gz -2 sample_R2.fastq.gz \
       -S sample.sam
```

### 4.5 Сравнение STAR vs HISAT2

| Параметр | STAR | HISAT2 |
|----------|------|--------|
| **Скорость** | Очень быстро | Медленнее |
| **Память** | Много | Экономно |
| **Лучше для** | Большие проекты | Малые проекты, ограничения памяти |

### 4.6 Псевдовыравнивание (Salmon, Kallisto)

#### Преимущества

✅ Работает с транскриптомом (без сплайсинга)
✅ На порядки быстрее (минуты вместо часов)
✅ Меньше памяти

#### Индексация транскриптома (Salmon)

```bash
salmon index -t transcripts.fa -i salmon_index
```

#### Квантификация

```bash
salmon quant -i salmon_index \
             -l A \
             -1 sample_R1.fastq.gz -2 sample_R2.fastq.gz \
             -o sample_quant \
             --gcBias --seqBias
```

- `-l A` — автоопределение типа библиотеки
- `--gcBias` — коррекция на GC
- `--seqBias` — коррекция позиционных ошибок

### 4.7 Форматы выходных файлов

| Инструмент | Выход | Формат |
|-----------|-------|--------|
| **STAR** | ReadsPerGene.out.tab | Подсчёт по генам |
| **featureCounts** | counts.txt | Универсальный подсчёт из BAM |
| **HTSeq-count** | counts.txt | Аналогично featureCounts |
| **Salmon/Kallisto** | quant.sf | TPM оценки |

## Часть 5: Сборка транскриптомов de novo (Trinity)

### 5.1 Когда нужна сборка de novo?

✅ Референсный геном неизвестен (новый организм)
✅ Геном низкого качества (много пробелов)
✅ Трудный геном (большой, с повторами)

### 5.2 Отличия от геномной сборки

- Неравномерное покрытие (высокоэкспрессирующиеся гены больше)
- Наличие изоформ (один ген → несколько транскриптов)
- Направленность (РНК с одной цепи)

### 5.3 Trinity — сборка транскриптомов

#### Три этапа

| Этап | Описание |
|------|---------|
| **Inchworm** | Линейные контиги из k-меров (жадный алгоритм) |
| **Chrysalis** | Группировка контигов в кластеры |
| **Butterfly** | Граф де Брёйна и восстановление изоформ |

#### Установка

```bash
conda install -c bioconda trinity
```

#### Запуск

```bash
Trinity --seqType fq \
        --left sample_R1.fastq.gz \
        --right sample_R2.fastq.gz \
        --CPU 20 \
        --max_memory 50G \
        --output trinity_out
```

#### Выходные файлы

- `trinity_out.Trinity.fasta` — все транскрипты (изоформы)
- `trinity_out.Trinity.fasta.gene_trans_map` — соответствие транскрипт-ген

### 5.4 Оценка качества сборки

- **BUSCO** — полнота по консервативным генам
- **TransRate** — метрики качества
- **N50, L50** — как в геномных сборках

## Часть 6: Количественная оценка экспрессии

### 6.1 Нормализация — зачем?

- Гены разной длины → разное количество ридов
- Разная глубина секвенирования между образцами
- Технические артефакты (batch, GC-состав)

### 6.2 Метрики нормализации

| Метрика | Формула | Назначение |
|---------|---------|-----------|
| **RPKM** | (риды/ген × 10⁹) / (длина × всего ридов) | Одноцепочечные |
| **FPKM** | (фрагменты/ген × 10⁹) / (длина × всего) | Парные |
| **TPM** | (риды/длина) / Σ(риды_i/длина_i) × 10⁶ | Лучше для сравнения |
| **DESeq2** | Медианное отношение к геом. среднему | Для DE-анализа |
| **CPM** | (риды/ген / всего) × 10⁶ | Простая нормализация |

### 6.3 Почему TPM лучше RPKM?

- В RPKM сумма зависит от состава образца
- В TPM сумма = 10⁶, позволяет сравнивать пропорции внутри образца

### 6.4 Подсчёт ридов на ген

#### Из BAM с featureCounts

```bash
featureCounts -a annotation.gtf -o counts.txt -T 20 -p -s 2 sample_sorted.bam
```

- `-s 2` — направленная библиотека (1 = прямая, 2 = обратная)

#### Из BAM с htseq-count

```bash
htseq-count -f bam -r pos -s reverse sample_sorted.bam annotation.gtf > counts.txt
```

#### Из Salmon/Kallisto

Уже на выходе в формате TPM

## Контрольные вопросы

**1. Принципиальное отличие выравнивания РНК от ДНК?**

**Ответ:** РНК-риды могут пересекать границы экзон-экзон (сплайс-сайты), поэтому нужны сплайс-чувствительные выравниватели (STAR, HISAT2).

**2. Методы обогащения по мРНК?**

**Ответ:** PolyA selection (magnet beads + oligo-dT, ~70% на экзоны) и RiboZero (удаление рРНК, ~40% на экзоны).

**3. Batch effect и минимизация?**

**Ответ:** Рандомизировать образцы, один лот реактивов, один человек, включить batch в модель.

**4. Биологические реплики?**

**Ответ:** Нужны для статистической мощности. Минимум 3 на условие (важнее чем больше ридов).

**5. Выравниватели РНК?**

**Ответ:** STAR (быстро, много памяти), HISAT2 (баланс), Salmon/Kallisto (псевдовыравнивание, очень быстро).

**6. Псевдовыравнивание?**

**Ответ:** Salmon/Kallisto работают с транскриптомом без сплайсинга. Быстро и экономно по памяти.

**7. Trinity — три этапа?**

**Ответ:** Inchworm (контиги), Chrysalis (кластеры), Butterfly (изоформы).

**8. RPKM vs FPKM vs TPM?**

**Ответ:** RPKM для одноцепочечных, FPKM для парных, TPM лучше (сумма = 10⁶).

**9. Нормализация перед DE?**

**Ответ:** Используйте сырые counts, DESeq2/edgeR сами нормализуют.

**10. Направленная библиотека?**

**Ответ:** Направленность определяется методом подготовки (dUTP). Влияет на подсчёт (параметр -s в featureCounts).

**11. UMI?**

**Ответ:** Случайные баркоды в адаптерах → отличают молекулы мРНК от ПЦР-дубликатов.

**12. Важные графики FastQC для RNA-seq?**

**Ответ:** GC-состав (унимодальное ли?), per base quality, adapter content.

**13. Метрики STAR лога?**

**Ответ:** % уникально выровненных ридов, % мультимаппингов, % неприлегающих.

**14. Когда нужна сборка de novo?**

**Ответ:** Неизвестный геном, низкое качество референса, сложный геном.

**15. Изоформа?**

**Ответ:** Разные варианты сплайсинга одного гена → разные мРНК/белки. Trinity восстанавливает через граф де Брёйна.
