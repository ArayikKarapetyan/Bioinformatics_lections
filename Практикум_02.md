## Часть 1. Знакомство с Conda и окружениями

Conda — это менеджер пакетов и окружений. Он позволяет создавать изолированные среды с нужными версиями Python/R и библиотек.

### 1.1. Проверка Conda

Подключитесь к серверу и выполните:

```bash
conda --version
```

Если вы видите версию (например, `conda 23.1.0`), значит Conda установлена. Если нет – обратитесь к администратору или установите Miniconda самостоятельно (инструкция есть в лекции).

> **Обратите внимание** на приглашение командной строки: обычно оно начинается с `(base)`, что означает, что активно базовое окружение.

### 1.2. Просмотр списка окружений

```bash
conda env list
```

или

```bash
conda info --envs
```

Вы увидите список всех созданных окружений. Сейчас, скорее всего, только `base`.

### 1.3. Создание нового пустого окружения

Создадим окружение с именем `bioinfo_practice`:

```bash
conda create -n bioinfo_practice
```

Подтвердите создание, нажав `y`. После создания окружение появится в списке.

### 1.4. Активация и деактивация окружения

Активируйте новое окружение:

```bash
conda activate bioinfo_practice
```

Приглашение должно измениться на `(bioinfo_practice)`. Теперь все команды и устанавливаемые пакеты будут относиться только к этому окружению.

Выйдите из окружения:

```bash
conda deactivate
```

Снова вернулись в `(base)`.

### 1.5. Создание окружения с конкретной версией Python и пакетами

Создадим окружение `py39` с Python 3.9 и несколькими популярными библиотеками:

```bash
conda create -n py39 python=3.9 pandas numpy jupyter ipykernel
```

Подтвердите установку. Это займёт некоторое время: Conda разрешит зависимости и скачает пакеты.

### 1.6. Активация нового окружения и проверка

```bash
conda activate py39
python --version          # должно показать Python 3.9.x
pip list                  # посмотреть установленные пакеты (pandas, numpy и т.д.)
```

### 1.7. Установка дополнительного пакета

Находясь в окружении `py39`, установим, например, библиотеку `seaborn` для графиков:

```bash
conda install seaborn
```

> **Примечание:** Или можно использовать `pip`, но лучше стараться использовать `conda`, если пакет есть в каналах Conda.

### 1.8. Экспорт окружения в YAML-файл

Окружением можно поделиться с коллегами. Экспортируем текущее окружение в файл:

```bash
conda env export > py39_environment.yaml
```

Посмотрите содержимое файла:

```bash
cat py39_environment.yaml
```

Вы увидите полный список всех пакетов с версиями.

### 1.9. Создание точной копии окружения из YAML (имитация)

Для тренировки удалим текущее окружение (не бойтесь, мы его восстановим):

```bash
conda deactivate          # сначала выйти из окружения, если мы в нём
conda remove -n py39 --all
```

Подтвердите удаление. Проверьте, что его нет: `conda env list`.

Теперь восстановим окружение из файла:

```bash
conda env create -f py39_environment.yaml
```

Это создаст окружение с тем же именем (указано в YAML) и теми же пакетами. Если имя не указано в файле, можно задать через `-n новое_имя`.

### 1.10. Добавление дополнительных каналов (channels)

В биоинформатике часто нужны пакеты из каналов `conda-forge` и `bioconda`. Добавим их в конфигурацию:

```bash
conda config --add channels conda-forge
conda config --add channels bioconda
```

Посмотрим текущие каналы:

```bash
conda config --show channels
```

Каналы будут проверяться в порядке добавления (приоритет у последнего добавленного). Теперь можно устанавливать, например, биоинформатический пакет `fastqc`:

```bash
conda install -n py39 fastqc   # установим в окружение py39 (если оно активно, можно без -n)
```

### 1.11. Удаление окружения (если нужно)

```bash
conda remove -n bioinfo_practice --all
```

## Часть 2. Jupyter и работа с разными ядрами (kernels)

Jupyter позволяет использовать разные окружения как ядра для ноутбуков.

### 2.1. Регистрация окружения в Jupyter

Активируйте окружение `py39`:

```bash
conda activate py39
```

Установите `ipykernel`, если ещё не сделали (мы его установили при создании, но на всякий случай):

```bash
conda install ipykernel
```

Теперь зарегистрируем это окружение как ядро Jupyter:

```bash
python -m ipykernel install --user --name py39 --display-name "Python 3.9 (bioinfo)"
```

- `--name` – внутреннее имя ядра (должно совпадать с именем окружения)
- `--display-name` – как оно будет отображаться в интерфейсе Jupyter

### 2.2. Запуск Jupyter Hub (на сервере)

Если у вас есть доступ к Jupyter Hub (например, `https://juphub.aldan3.itm-rsmu.ru`), войдите под своим логином/паролем. При запуске сервера выберите ресурсы (например, `default`). После запуска откроется интерфейс Jupyter Lab.

Создайте новый ноутбук: нажмите кнопку `+` и в разделе Notebook выберите ядро "Python 3.9 (bioinfo)".

### 2.3. Проверка ядра

В первой ячейке ноутбука выполните:

```python
import sys
print(sys.executable)   # покажет путь к Python из вашего окружения
import pandas as pd
print(pd.__version__)
```

Если всё правильно, вы увидите путь, содержащий `envs/py39/`, и версию pandas.

### 2.4. Горячие клавиши в Jupyter Lab

Попрактикуйтесь в использовании шорткатов из лекции (режим командный/редактирования):

| Клавиша | Действие |
|---------|----------|
| `Esc` | Командный режим |
| `Enter` | Режим редактирования |
| `A` | Вставить ячейку выше |
| `B` | Вставить ячейку ниже |
| `D D` (дважды) | Удалить ячейку |
| `Shift + Enter` | Выполнить ячейку и перейти к следующей |
| `Ctrl + Enter` | Выполнить текущую |
| `M` | Преобразовать ячейку в Markdown |
| `Y` | Преобразовать в код |
| `Tab` | Автодополнение |

### 2.5. Оформление ноутбука Markdown

Создайте ячейку Markdown и напишите заголовок:

```markdown
# Мой первый анализ
## Подготовка данных
*   Список библиотек:
    *   pandas
    *   numpy
```

Выполните ячейку (`Shift+Enter`), чтобы увидеть отформатированный текст.

### 2.6. Запуск bash-команд в Jupyter

Можно прямо из ноутбука выполнять команды Linux двумя способами:

С восклицательным знаком:
```python
!ls -la
```

Магическая команда `%%bash` в начале ячейки:
```python
%%bash
echo "Текущая директория:"
pwd
ls -lh
```

## Часть 3. Основы Git

Git поможет отслеживать изменения в ваших скриптах и файлах.

### 3.1. Настройка Git (один раз)

Укажите своё имя и email (они будут привязаны к коммитам):

```bash
git config --global user.name "Ваше Имя Фамилия"
git config --global user.email "ваш_email@example.com"
```

Полезно также установить редактор по умолчанию (например, nano):

```bash
git config --global core.editor "nano -w"
```

Посмотреть все настройки:

```bash
git config --global --list
```

### 3.2. Создание репозитория

Создайте папку для проекта и перейдите в неё:

```bash
mkdir ~/my_first_repo
cd ~/my_first_repo
```

Инициализируйте Git-репозиторий:

```bash
git init
```

Появится скрытая папка `.git` (проверьте `ls -a`).

### 3.3. Первый коммит

Создайте файл `README.md`:

```bash
echo "# Мой первый проект по биоинформатике" > README.md
```

Проверьте статус:

```bash
git status
```

Вы увидите, что файл `README.md` не отслеживается (untracked). Добавьте его в индекс (staging area):

```bash
git add README.md
```

Снова `git status` – файл теперь зелёный, готов к коммиту.

Выполните коммит с сообщением:

```bash
git commit -m "Initial commit: add README.md"
```

### 3.4. Второй коммит (изменение файла)

Добавьте ещё одну строку в README:

```bash
echo "Этот проект посвящён анализу RNA-seq данных." >> README.md
```

Проверьте статус – файл изменён (modified). Добавьте его в индекс и сделайте коммит:

```bash
git add README.md
git commit -m "Add project description"
```

### 3.5. Просмотр истории

```bash
git log
```

Вы увидите список коммитов с хешами, автором, датой и сообщением. Выйдите из просмотра, нажав `q`.

### 3.6. Создание .gitignore (полезно для больших данных)

Создайте файл `.gitignore` и укажите в нём шаблоны файлов, которые не нужно отслеживать (например, временные файлы, данные):

```bash
echo "*.fastq" >> .gitignore
echo "*.bam" >> .gitignore
echo "data/" >> .gitignore
```

Добавьте `.gitignore` в репозиторий:

```bash
git add .gitignore
git commit -m "Add gitignore for bioinformatics data files"
```

### 3.7. Работа с удалённым репозиторием (если есть доступ к GitLab)

На сервере может быть локальный GitLab. Если у вас есть логин, можно создать там пустой репозиторий и связать его:

```bash
git remote add origin https://gitlab.вашсервер.ru/вашлогин/my_first_repo.git
git push -u origin master
```

Если доступа нет, пропустите этот шаг.

## Часть 4. Работа с архивами (zip, gzip)

Биологические данные часто сжаты для экономии места.

### 4.1. Подготовка файлов

Создадим несколько текстовых файлов для экспериментов:

```bash
mkdir ~/archive_practice
cd ~/archive_practice
seq 1 1000 > numbers.txt
echo -e ">seq1\nATCGATCGATCG\n>seq2\nGCTAGCTAGCTA" > sequences.fasta
echo "Sample,Value" > data.csv
for i in {1..10}; do echo "sample$i,$RANDOM" >> data.csv; done
```

Посмотрим, что получилось:

```bash
ls -lh
```

### 4.2. Работа с zip

**Создание zip-архива** всех csv-файлов (если бы они были, но у нас один):

```bash
zip data_archive.zip *.csv
```

**Добавление файла** в существующий архив:

```bash
zip data_archive.zip numbers.txt
```

**Просмотр содержимого** архива:

```bash
unzip -l data_archive.zip
```

или

```bash
less data_archive.zip   # может показать бинарные данные, но иногда работает
```

**Распаковка** архива:

```bash
unzip data_archive.zip -d extracted_zip
```

Посмотрите, что в папке `extracted_zip`.

**Удаление файла** из архива:

```bash
zip -d data_archive.zip numbers.txt
```

Проверьте содержимое снова.

### 4.3. Работа с gzip

gzip сжимает только один файл, заменяя его на сжатую версию (с расширением `.gz`).

**Сжатие файла:**

```bash
gzip sequences.fasta
```

Теперь файл называется `sequences.fasta.gz`, оригинал удалён.

**Просмотр содержимого** сжатого файла без распаковки:

```bash
zcat sequences.fasta.gz
```

> Если `zcat` не работает, попробуйте `gunzip -c sequences.fasta.gz`.

**Распаковка:**

```bash
gunzip sequences.fasta.gz
```

Файл снова `sequences.fasta`.

**Распаковка с сохранением сжатой копии:**

```bash
gzip -k sequences.fasta          # создаст sequences.fasta.gz, оставив оригинал
gunzip -k sequences.fasta.gz     # распакует, оставив .gz
```

### 4.4. Комбинация tar и gzip (tar.gz)

Часто встречаются архивы `.tar.gz` – это сначала объединение в один файл (tar), а потом сжатие (gzip).

**Создание tar.gz** (если хотите запаковать папку целиком):

```bash
tar -czf practice_archive.tar.gz extracted_zip/ numbers.txt data.csv
```

- `-c` – создать архив
- `-z` – сжать через gzip
- `-f` – имя файла

**Просмотр содержимого** tar.gz:

```bash
tar -tzf practice_archive.tar.gz
```

**Распаковка:**

```bash
tar -xzf practice_archive.tar.gz -C extracted_tar
```

## Часть 5. Права доступа к файлам и bash-скрипты

### 5.1. Понимание вывода ls -l

Выполните в любой папке:

```bash
ls -l
```

Вы увидите строки вроде:

```
-rw-r--r-- 1 user group 1234 Mar 2 15:00 file.txt
drwxr-xr-x 2 user group 4096 Mar 2 15:00 folder
```

**Разберём первую часть:**

- **Первый символ:** `-` (файл), `d` (директория), `l` (ссылка)
- **Следующие 9 символов** делятся на три группы по три:
  - **Владелец (user):** `rw-` (чтение, запись, нет исполнения)
  - **Группа (group):** `r--` (только чтение)
  - **Остальные (others):** `r--` (только чтение)

### 5.2. Изменение прав с помощью chmod

Создадим скрипт и сделаем его исполняемым.

```bash
nano hello.sh
```

Напишите внутри:

```bash
#!/bin/bash
echo "Привет, мир!"
echo "Сегодня `date`"
```

Сохраните (`Ctrl+O`, `Enter`) и выйдите (`Ctrl+X`).

Посмотрите права:

```bash
ls -l hello.sh
```

Скорее всего: `-rw-r--r--` – нет права на исполнение (`x`).

Добавим право на исполнение для владельца:

```bash
chmod u+x hello.sh
```

Теперь права: `-rwxr--r--`. Можно запустить скрипт:

```bash
./hello.sh
```

Если хотим дать право на исполнение всем (владельцу, группе, остальным):

```bash
chmod +x hello.sh    # (без u/g/o – добавляет всем)
```

**Числовой способ:** `chmod 755 hello.sh` даст `rwxr-xr-x`. (7=rwx, 5=r-x).

### 5.3. Простой bash-скрипт с циклом

Создайте скрипт `process.sh`:

```bash
#!/bin/bash
# Скрипт для создания резервных копий файлов
for file in *.csv; do
    cp "$file" "backup_$file"
    echo "Скопирован $file"
done
```

Сделайте его исполняемым и запустите:

```bash
chmod +x process.sh
./process.sh
```

Проверьте, появились ли файлы с префиксом `backup_`.

## Часть 6. Задания для самостоятельной работы

1. **Conda:** Создайте окружение с именем `r_env`, установите в него R и пакет `tidyverse` (подсказка: `conda create -n r_env r-base r-tidyverse`). Активируйте его и запустите R (`R`), проверьте, что `library(tidyverse)` работает. Выйдите из R (`q()`).

2. **Jupyter:** Зарегистрируйте окружение `r_env` как ядро для R (нужен пакет `irkernel`; установите его в окружении и выполните `R -e "IRkernel::installspec(name='r_env', displayname='R (tidyverse)')"`). Затем в Jupyter Hub создайте ноутбук с этим ядром и выполните простую команду, например `plot(1:10)`.

3. **Git:** В папке `~/my_first_repo` создайте новый файл `analysis.R` с любым содержимым (например, комментарий). Добавьте его в репозиторий и сделайте коммит с сообщением "Add R script". Затем измените файл, сделайте ещё один коммит. Посмотрите историю (`git log --oneline`).

4. **Архивы:** Скачайте любой небольшой файл с помощью `wget` (например, `wget https://ftp.ncbi.nlm.nih.gov/pub/pmc/PMC-ids.csv.gz`). Распакуйте его, посмотрите первые 5 строк распакованного файла, затем снова запакуйте. Удалите распакованный файл.

5. **Права доступа:** Создайте скрипт `cleanup.sh`, который удаляет все файлы с расширением `.tmp` в текущей папке. Сделайте его исполняемым, но сначала создайте несколько пустых `.tmp` файлов для теста (`touch test1.tmp test2.tmp`). Запустите скрипт и убедитесь, что файлы удалены.
