## Часть 1: Зачем нужен тримминг?

### 1.1 Проблемы сырых ридов

| Проблема | Описание | Решение |
|----------|---------|--------|
| **Адаптеры** | Если фрагмент < длины рида, секвенатор читает адаптер | Удалить адаптеры |
| **Низкое качество в конце** | Истощение реактивов, рассинхронизация фаз | TRAILING обрезка |
| **Плохое качество в начале** | Калибровка детекции | LEADING обрезка |
| **Загрязнения** | ДНК от других организмов, праймеры | Ручная проверка |
| **Слишком короткие риды** | После обрезки становятся непригодными | Отбросить (MINLEN) |

### 1.2 Что даёт тримминг?

✅ **Удалить адаптеры** — не мешают выравниванию
✅ **Отрезать низкокачество** — выше точность анализа
✅ **Отбросить короткие риды** — избежать ложных выравниваний
✅ **Унифицировать данные** — единый формат

### 1.3 Визуализация результата

**До:**
- Падение качества к концу
- Низкое качество в начале
- Присутствие адаптеров

**После:**
- Ровное качество
- Адаптеры удалены
- Риды короче, но качественнее

## Часть 3: Trimmomatic — подробный разбор

### 3.1 Общая структура команды

```bash
trimmomatic PE input_R1.fastq.gz input_R2.fastq.gz \
    -baseout output_prefix \
    -trimlog trim.log \
    ILLUMINACLIP:adapters.fa:2:30:10:2:keepBothReads \
    LEADING:3 \
    TRAILING:3 \
    SLIDINGWINDOW:4:20 \
    MINLEN:36
```

| Параметр | Описание |
|----------|---------|
| **PE** | Режим парных ридов (SE — одинарные) |
| **-baseout** | Префикс выходных файлов |
| **-trimlog** | Лог-файл о каждом обрезанном риде |

### 3.2 Параметры тримминга

#### ILLUMINACLIP — удаление адаптеров

```
ILLUMINACLIP:<fastaWithAdapters>:<seed mismatches>:<palindrome clip threshold>
:<simple clip threshold>:<minAdapterLength>:<keepBothReads>
```

| Параметр | Значение | Описание |
|----------|----------|---------|
| fastaWithAdapters | TruSeq3-PE.fa | Файл с адаптерами |
| seed mismatches | 2 | Несовпадений при поиске начального совпадения |
| palindrome threshold | 30 | Порог палиндромных адаптеров (оба рида через адаптер) |
| simple threshold | 10 | Порог простого обнаружения |
| minAdapterLength | 2 | Минимальная длина адаптера |
| keepBothReads | keepBothReads | Сохранять оба рида |

#### LEADING — обрезка начала

```
LEADING:<quality>
```

Отрезает основания с 5'-конца, пока качество < порога (обычно 3).

#### TRAILING — обрезка конца

```
TRAILING:<quality>
```

Отрезает основания с 3'-конца (обычно 3).

#### SLIDINGWINDOW — скользящее окно ⭐ **САМЫЙ ВАЖНЫЙ**

```
SLIDINGWINDOW:<windowSize>:<requiredQuality>
```

| Параметр | Значение | Описание |
|----------|----------|---------|
| windowSize | 4 | Размер окна в основаниях |
| requiredQuality | 20 | Минимальное среднее качество в окне |

**Алгоритм:** Движемся 5'→3', считаем среднее качество в окне. Когда падает ниже порога — отрезаем всё после.

**Проблема:** Если начало плохое, отрезает весь рид. Поэтому используйте LEADING перед SLIDINGWINDOW.

#### MAXINFO — адаптивный тримминг (целевая длина)

```
MAXINFO:<targetLength>:<strictness>
```

| Параметр | Значение | Описание |
|----------|----------|---------|
| targetLength | 50 | Целевая длина рида |
| strictness | 0-1 | 0 = важнее длина, 1 = важнее качество |

#### CROP и HEADCROP — принудительная обрезка

```
CROP:<length>           # Обрезать до длины (с конца)
HEADCROP:<length>       # Удалить N оснований с начала
```

#### MINLEN — минимальная длина

```
MINLEN:<length>
```

Риды короче этой длины после обрезок отбрасываются (обычно 36).

### 3.3 Работа с парными ридами

После тримминга получаются 4 файла:

| Файл | Описание |
|------|---------|
| **_1P.fastq.gz** | R1 с парой R2 (paired) |
| **_2P.fastq.gz** | R2 с парой R1 (paired) |
| **_1U.fastq.gz** | R1 без пары (unpaired) |
| **_2U.fastq.gz** | R2 без пары (unpaired) |

⚠️ **Большинство downstream-программ используют только парные риды!**

### 3.4 Практическое задание

```bash
cd ~/tutorials/trimming

trimmomatic PE raw/SRR2121774_1.fastq.gz raw/SRR2121774_2.fastq.gz \
    -baseout trimmed/SRR2121774 \
    -trimlog logs/SRR2121774.log \
    ILLUMINACLIP:adapters/TruSeq3-PE.fa:2:30:10:2:keepBothReads \
    LEADING:3 \
    TRAILING:3 \
    SLIDINGWINDOW:4:20 \
    MINLEN:36

# Результаты
ls -lh trimmed/

# FastQC после тримминга
fastqc trimmed/*P.fastq.gz -o fastqc_after/
multiqc fastqc_after/ -o multiqc_after/
```

## Часть 4: Параллельный запуск Trimmomatic для многих образцов

### 4.1 Использование GNU Parallel

```bash
# Создаём список образцов
ls raw/*_1.fastq.gz | sed 's/_1.fastq.gz//' | sed 's/raw\///' > samples.txt

# Параллельный запуск (4 процесса одновременно)
cat samples.txt | parallel -j4 "trimmomatic PE raw/{}_1.fastq.gz raw/{}_2.fastq.gz \
    -baseout trimmed/{} \
    -trimlog logs/{}.log \
    ILLUMINACLIP:adapters/TruSeq3-PE.fa:2:30:10:2:keepBothReads \
    LEADING:3 \
    TRAILING:3 \
    SLIDINGWINDOW:4:20 \
    MINLEN:36"
```

### 4.2 SLURM-скрипт для массового запуска

Создайте файл `run_trimmomatic.slurm`:

```bash
#!/bin/bash
#SBATCH --job-name=trimmomatic
#SBATCH --cpus-per-task=40
#SBATCH --mem=256gb
#SBATCH --time=1:30:00
#SBATCH --output=logs/trimmomatic_%j.log
#SBATCH --partition=short

cd /home/your_username/tutorials/trimming/trimmed

# Параллельный запуск для всех образцов
ls /home/your_username/tutorials/trimming/raw/*_1.fastq.gz | parallel -j4 \
    "trimmomatic PE {} {=s/_1/_2/=} \
    -baseout {=s:.*/::;s/_1.*//=} \
    -trimlog /home/your_username/tutorials/trimming/logs/{=s:.*/::;s/_1.*//=}.log \
    ILLUMINACLIP:/home/your_username/tutorials/trimming/adapters/TruSeq3-PE.fa:2:30:10:2:keepBothReads \
    LEADING:3 TRAILING:3 \
    SLIDINGWINDOW:4:20 MINLEN:36"
```

Отправка:

```bash
sbatch run_trimmomatic.slurm
squeue -u $USER
```

## Часть 5: Cutadapt — простой инструмент для удаления адаптеров

### 5.1 Основной синтаксис

```bash
cutadapt -a ADAPTER_FWD -A ADAPTER_REV \
    -o out.1.fastq.gz -p out.2.fastq.gz \
    reads.1.fastq.gz reads.2.fastq.gz
```

| Параметр | Описание |
|----------|---------|
| **-a** | Адаптер для R1 (3'-конец) |
| **-A** | Адаптер для R2 (3'-конец) |
| **-g/-G** | 5'-конец (начало) |
| **-o** | Выходной файл R1 |
| **-p** | Выходной файл R2 |

### 5.2 Пример

```bash
cutadapt -a AGATCGGAAGAGCACACGTCTGAACTCCAGTCA \
    -A AGATCGGAAGAGCGTCGTGTAGGGAAAGAGTGT \
    -o trimmed.R1.fastq.gz -p trimmed.R2.fastq.gz \
    raw.R1.fastq.gz raw.R2.fastq.gz
```

### 5.3 Применение

- Удаление адаптеров в данных MGI
- Удаление праймеров из ампликонных данных
- Переворот ридов в нужную ориентацию

## Часть 6: Fastp — современная быстрая альтернатива

### 6.1 Преимущества

✅ **Скорость** — написан на C++, очень быстрый
✅ **Простота** — много параметров уже настроено
✅ **Встроенный QC** — генерирует HTML-отчёт как FastQC
✅ **Автодетекция** — может сам найти адаптеры
✅ **Парные риды** — удобно работает

### 6.2 Базовый синтаксис

```bash
fastp -i in.R1.fastq.gz -I in.R2.fastq.gz \
    -o out.R1.fastq.gz -O out.R2.fastq.gz \
    --detect_adapter_for_pe \
    --cut_front --cut_tail --cut_right \
    --length_required 36 \
    --html report.html
```

### 6.3 Основные параметры

| Параметр | Описание |
|----------|---------|
| **-i/-I** | Входные файлы R1 и R2 |
| **-o/-O** | Выходные файлы R1 и R2 |
| **--detect_adapter_for_pe** | Автоматически искать адаптеры |
| **--cut_front** | LEADING (более умный) |
| **--cut_tail** | TRAILING (с окна) |
| **--cut_right** | SLIDINGWINDOW |
| **--length_required** | MINLEN |
| **--average_qual** | Минимальное среднее качество рида |
| **--html** | Сохранить отчёт |
| **--json** | Сохранить данные в JSON |

### 6.4 Пример

```bash
fastp -i raw/SRR2121774_1.fastq.gz -I raw/SRR2121774_2.fastq.gz \
    -o trimmed/SRR2121774_1.fastq.gz -O trimmed/SRR2121774_2.fastq.gz \
    --detect_adapter_for_pe \
    --cut_front --cut_tail --cut_right \
    --length_required 36 \
    --html reports/SRR2121774.html \
    --json reports/SRR2121774.json
```

### 6.5 Trimmomatic vs Fastp

| Аспект | Trimmomatic | Fastp |
|--------|------------|-------|
| **Скорость** | Медленнее | ⚡ Быстрее |
| **Контроль** | ✅ Много параметров | Меньше опций |
| **Простота** | Сложнее | ✅ Проще |
| **QC отчёт** | Нет | ✅ Встроенный |
| **Рекомендация** | Стандарт пайплайнов | Для большинства задач |

## Часть 7: Менеджеры пайплайнов — Snakemake и Nextflow

### 7.1 Зачем нужны менеджеры пайплайнов?

**Проблемы без пайплайнов:**
- Как запустить все этапы в правильном порядке?
- Как распараллелить обработку множества образцов?
- Что если один этап упал? Перезапускать всё заново?
- Как обеспечить воспроизводимость?
- Как поделиться пайплайном с коллегами?

**Решения менеджеров пайплайнов:**
✅ Автоматизация — опишите один раз, запустите много раз
✅ Параллелизация — распределение по ядрам/узлам
✅ Отслеживание зависимостей — перезапускают только нужное
✅ Воспроизводимость — пайплайн + конфиг = полное описание
✅ Интеграция с SLURM — отправка задач в очередь

### 7.2 Snakemake vs Nextflow

| Аспект | Snakemake | Nextflow |
|--------|-----------|----------|
| **Язык** | Python-подобный | Groovy (Java-подобный) |
| **Популярность** | Очень популярен в академии | Становится стандартом индустрии |
| **Готовые пайплайны** | Много отдельных | nf-core — огромная коллекция |
| **Интеграция SLURM** | Есть | ⭐ Отличная |
| **Кривая обучения** | Проще для питонистов | Крутая кривая |

### 7.3 Основные концепции Snakemake

#### Правило (rule)

```python
rule fastqc:
    input:
        "raw/{sample}_1.fastq.gz",
        "raw/{sample}_2.fastq.gz"
    output:
        "fastqc/{sample}_1_fastqc.html",
        "fastqc/{sample}_2_fastqc.html"
    shell:
        "fastqc {input} -o fastqc/"
```

- `{sample}` — подстановка (wildcard)
- `input` — входные файлы
- `output` — выходные файлы
- `shell` — команда

#### Целевое правило (rule all)

```python
rule all:
    input:
        expand("trimmed/{sample}_1P.fastq.gz", sample=SAMPLES),
        expand("trimmed/{sample}_2P.fastq.gz", sample=SAMPLES),
        expand("fastqc_after/{sample}_1P_fastqc.html", sample=SAMPLES)
```

#### Запуск

```bash
snakemake -s Snakefile --cores 40

# Пробный запуск (показать, что будет сделано)
snakemake -s Snakefile --dry-run --print

# Только до определённого правила
snakemake --until trimmomatic

# Перезапустить незавершённые
snakemake --rerun-incomplete

# Визуализировать граф зависимостей
snakemake --dag | dot -T svg > dag.svg

# Создать отчёт
snakemake --report report.html
```

### 7.4 Пример полного Snakemake-пайплайна

**Snakefile:**

```python
import glob

configfile: "config.yaml"

SAMPLES = [s.replace("raw/", "").replace("_1.fastq.gz", "") 
           for s in glob.glob("raw/*_1.fastq.gz")]

rule all:
    input:
        expand("trimmed/{sample}_1P.fastq.gz", sample=SAMPLES),
        expand("trimmed/{sample}_2P.fastq.gz", sample=SAMPLES),
        expand("fastqc_trimmed/{sample}_1P_fastqc.html", sample=SAMPLES),
        "multiqc/multiqc_report.html"

rule fastqc_raw:
    input:
        r1="raw/{sample}_1.fastq.gz",
        r2="raw/{sample}_2.fastq.gz"
    output:
        "fastqc_raw/{sample}_1_fastqc.html",
        "fastqc_raw/{sample}_2_fastqc.html"
    shell:
        "fastqc {input.r1} {input.r2} -o fastqc_raw/"

rule trimmomatic:
    input:
        r1="raw/{sample}_1.fastq.gz",
        r2="raw/{sample}_2.fastq.gz"
    output:
        r1p="trimmed/{sample}_1P.fastq.gz",
        r2p="trimmed/{sample}_2P.fastq.gz"
    params:
        adapters=config["adapters"],
        leading=config.get("leading", 3),
        trailing=config.get("trailing", 3),
        window=config.get("slidingwindow", "4:20"),
        minlen=config.get("minlen", 36)
    shell:
        "trimmomatic PE {input.r1} {input.r2} "
        "-baseout trimmed/{wildcards.sample} "
        "ILLUMINACLIP:{params.adapters}:2:30:10:2:keepBothReads "
        "LEADING:{params.leading} TRAILING:{params.trailing} "
        "SLIDINGWINDOW:{params.window} MINLEN:{params.minlen}"

rule fastqc_trimmed:
    input:
        r1p="trimmed/{sample}_1P.fastq.gz",
        r2p="trimmed/{sample}_2P.fastq.gz"
    output:
        "fastqc_trimmed/{sample}_1P_fastqc.html",
        "fastqc_trimmed/{sample}_2P_fastqc.html"
    shell:
        "fastqc {input.r1p} {input.r2p} -o fastqc_trimmed/"

rule multiqc:
    input:
        expand("fastqc_trimmed/{sample}_{read}_fastqc.html", 
               sample=SAMPLES, read=['1P','2P'])
    output:
        "multiqc/multiqc_report.html"
    shell:
        "multiqc fastqc_trimmed -o multiqc/"
```

**config.yaml:**

```yaml
adapters: "/home/user/tutorials/trimming/adapters/TruSeq3-PE.fa"
leading: 3
trailing: 3
slidingwindow: "4:20"
minlen: 36
```

## Контрольные вопросы

**1. Почему в ридах появляются адаптерные последовательности?**

**Ответ:** Если фрагмент ДНК короче длины прочтения, секвенатор продолжает читать после конца фрагмента и захватывает адаптерную последовательность.

**2. Как работает SLIDINGWINDOW? В чём недостаток?**

**Ответ:** Движется 5'→3', считает среднее качество в окне. Когда падает ниже порога — отрезает всё. Недостаток: если качество низкое в начале, отрезает весь рид. Решение: LEADING перед SLIDINGWINDOW.

**3. Чем LEADING отличается от HEADCROP?**

**Ответ:** LEADING удаляет основания с начала, пока качество < порога (адаптивно). HEADCROP принудительно удаляет N оснований с начала (фиксированное).

**4. Что такое _1P и _1U файлы?**

**Ответ:** _1P — R1 с парой (paired), _1U — R1 без пары (unpaired). Большинство downstream-программ используют только paired.

**5. Какой параметр отвечает за минимальную длину?**

**Ответ:** MINLEN — риды короче этой длины отбрасываются.

**6. Преимущества fastp перед Trimmomatic?**

**Ответ:** Скорость (C++), простота, встроенный HTML-отчёт, автодетекция адаптеров.

**7. Как задать адаптеры в Cutadapt?**

**Ответ:** `-a` для R1 (3'-конец), `-A` для R2. Можно также `-g/-G` для 5'-конца.

**8. Зачем нужны менеджеры пайплайнов?**

**Ответ:** Автоматизация, параллелизация, воспроизводимость, управление зависимостями, интеграция с кластером.

**9. Snakemake vs Nextflow?**

**Ответ:** Snakemake проще для питонистов, Nextflow более стандартизирован для индустрии. nf-core имеет готовые пайплайны.

**10. Что такое rule all в Snakemake?**

**Ответ:** Целевое правило, указывает, какие файлы должны быть получены в итоге. Snakemake работает в обратном направлении от этих файлов.

**11. Как в Snakemake обрабатывать множество образцов?**

**Ответ:** Используются wildcards ({sample}) и expand() для генерации списков файлов.

**12. Что такое nf-core?**

**Ответ:** Сообщество готовых, проверенных пайплайнов на Nextflow для стандартных задач (RNA-seq, ChIP-seq и др.).

**13. Как организовать пайплайн для 100 образцов на кластере?**

**Ответ:** Использовать Snakemake или Nextflow с параллелизацией через SLURM, параметры в конфиг-файле, все образцы обрабатываются в одной команде.
