## Часть 1: От сборки de novo к ресеквенированию

### 1.1 Два подхода

| Подход | Применение | Сложность |
|--------|-----------|----------|
| **Сборка de novo** | Новые, неизученные организмы | Сложная |
| **Ресеквенирование** | Известный референсный геном | Простая и быстрая |

### 1.2 Зачем нужно ресеквенирование?

✅ Поиск генетических вариантов и заболеваний
✅ Популяционная генетика и разнообразие
✅ Персонализированная медицина
✅ Фармакогенетика (метаболизм лекарств)

### 1.3 Общий пайплайн анализа

```
FASTQ (сырые риды)
    ↓
Тримминг (fastp, Trimmomatic)
    ↓
Выравнивание на референс (BWA)
    ↓
Пост-обработка (сортировка, индексация, удаление дубликатов)
    ↓
Поиск вариантов (GATK, bcftools)
    ↓
VCF (файл с вариантами)
    ↓
Аннотация (SnpEff, ANNOVAR)
    ↓
Интерпретация врачом-генетиком
```

## Часть 3: Выравнивание на референс — BWA

### 3.1 Burrows-Wheeler Aligner (BWA)

**Сайт:** https://github.com/lh3/bwa

**Алгоритм:** преобразование Барроуза-Уилера (BWT) и FM-индекс

### 3.2 Запуск BWA MEM (для ридов 70bp+)

```bash
bwa mem -t 40 \
    /projects/mipt_dbmp_biotechnology/GRCh38/Homo_sapiens.GRCh38.dna.primary_assembly.fa \
    sample_trimmed_R1.fastq.gz \
    sample_trimmed_R2.fastq.gz | gzip -3 > aln_pe.sam.gz
```

| Параметр | Описание |
|----------|---------|
| `-t 40` | 40 потоков |
| Выход | aln_pe.sam.gz (сжатый SAM) |

### 3.3 Формат SAM (Sequence Alignment/Map)

**Пример строки:**

```
r001   99   chr1   7   30   8M2I4M1D3M   =   37   39   TTAGATAAAGGATACTG   *
```

#### Основные поля

| Поле | Описание |
|------|---------|
| QNAME | Идентификатор рида |
| FLAG | Битовая маска (парность, направление, выравнивание) |
| RNAME | Имя хромосомы |
| POS | Позиция начала (1-индексированная) |
| MAPQ | Качество выравнивания (выше = лучше) |
| CIGAR | Строка выравнивания (8M2I4M1D3M) |
| RNEXT | Хромосома парного рида |
| PNEXT | Позиция парного рида |
| TLEN | Длина вставки |
| SEQ | Последовательность рида |
| QUAL | Качества (Phred+33) |

#### FLAG (полезные значения)

| Значение | Описание |
|----------|---------|
| 1 | Рид парный |
| 2 | Оба рида выровнены правильно |
| 4 | Рид не выровнен |
| 16 | Выровнен на обратную цепь |
| 64 | Первый рид в паре |
| 128 | Второй рид в паре |

#### CIGAR (операции выравнивания)

| Операция | Описание |
|----------|---------|
| **M** | Совпадение/несовпадение |
| **I** | Вставка в рид |
| **D** | Делеция из рида |
| **N** | Пропуск (интрон) |
| **S** | Мягкая обрезка (есть в риде, но не выровнена) |
| **H** | Жёсткая обрезка (нет в риде) |

### 3.4 Конвертация SAM → BAM и сортировка

#### Почему BAM?

- Бинарный формат (меньше памяти)
- Быстрее обрабатывается
- Требуется для downstream-инструментов

#### Конвертация с samtools

```bash
# Распаковать SAM
gunzip -c aln_pe.sam.gz > aln_pe.sam

# Конвертировать в BAM и сортировать
samtools sort -@ 40 -o sample_sorted.bam aln_pe.sam

# Индексировать (создастся sample_sorted.bam.bai)
samtools index sample_sorted.bam
```

#### В одну команду (без промежуточного файла)

```bash
bwa mem -t 40 reference.fa read1.fq read2.fq | samtools sort -@ 40 -o sample_sorted.bam
```

#### Просмотр BAM

```bash
samtools view -h sample_sorted.bam | head -20
```

#### Статистика выравнивания

```bash
samtools flagstat sample_sorted.bam
```

## Часть 4: Пост-обработка BAM-файлов

### 4.1 Добавление информации о чтениях (RG - Read Group)

```bash
# Через samtools
samtools addreplacerg -r 'ID:sample1' -r 'SM:sample1' -r 'LB:lib1' -r 'PL:ILLUMINA' \
    -o sample_tagged.bam sample_sorted.bam

# Или через GATK
gatk AddOrReplaceReadGroups \
    I=sample_sorted.bam \
    O=sample_tagged.bam \
    RGID=sample1 \
    RGLB=lib1 \
    RGPL=ILLUMINA \
    RGPU=unit1 \
    RGSM=sample1
```

### 4.2 Удаление ПЦР-дубликатов

ПЦР-дубликаты — риды от одной исходной молекулы (искажают частоты аллелей)

```bash
gatk MarkDuplicates \
    I=sample_tagged.bam \
    O=sample_dedup.bam \
    M=sample_dedup_metrics.txt \
    REMOVE_DUPLICATES=true

# Переиндексировать
samtools index sample_dedup.bam
```

### 4.3 Базовая коррекция качества (BQSR)

```bash
# Создать таблицу ковариации
gatk BaseRecalibrator \
    -R reference.fa \
    -I sample_dedup.bam \
    --known-sites known_sites.vcf \
    -O sample_recal.table

# Применить коррекцию
gatk ApplyBQSR \
    -R reference.fa \
    -I sample_dedup.bam \
    --bqsr-recal-file sample_recal.table \
    -O sample_recal.bam
```

⚠️ Для учебных целей этот шаг можно пропустить

## Часть 5: Поиск вариантов (Variant Calling)

### 5.1 Типы вариантов

| Тип | Описание |
|-----|---------|
| **SNP** | Замена одного нуклеотида |
| **Индел** | Вставка или делеция (короткая) |
| **SV** | Структурный вариант (крупные перестройки) |

### 5.2 Инструменты для поиска вариантов

| Инструмент | Описание |
|-----------|---------|
| **GATK** | Стандарт индустрии (Broad Institute) |
| **FreeBayes** | Байесовский вариантколлер |
| **bcftools** | Часть samtools |
| **DeepVariant** | На основе нейросетей (Google) |
| **Strelka2** | Оптимизирован для опухолей |

### 5.3 Запуск GATK HaplotypeCaller

#### Для одного образца

```bash
gatk HaplotypeCaller \
    -R reference.fa \
    -I sample_recal.bam \
    -O sample_variants.vcf
```

#### Для многих образцов (GVCF режим)

```bash
# Для каждого образца создать GVCF
gatk HaplotypeCaller \
    -R reference.fa \
    -I sample1_recal.bam \
    -O sample1.g.vcf.gz \
    -ERC GVCF

# Объединить
gatk CombineGVCFs \
    -R reference.fa \
    --variant sample1.g.vcf.gz \
    --variant sample2.g.vcf.gz \
    -O cohort.g.vcf.gz

# Генотипировать
gatk GenotypeGVCFs \
    -R reference.fa \
    -V cohort.g.vcf.gz \
    -O cohort_variants.vcf
```

### 5.4 Формат VCF (Variant Call Format)

#### Структура VCF

```
##fileformat=VCFv4.2
##INFO=<ID=DP,Number=1,Type=Integer,Description="Total Depth">
#CHROM POS ID REF ALT QUAL FILTER INFO FORMAT sample
1      12   .   G   A   40   PASS   DP=13;AF=0.5   GT:GQ   0/1:30
```

#### Основные поля

| Поле | Описание |
|------|---------|
| CHROM | Хромосома |
| POS | Позиция (1-индексированная) |
| ID | Идентификатор (rsID из dbSNP) |
| REF | Референсный аллель |
| ALT | Альтернативный аллель |
| QUAL | Качество вызова |
| FILTER | PASS или причина отбраковки |
| INFO | Дополнительная информация |
| FORMAT | Формат генотипа |
| sample | Генотип (0/1 = гетерозигота, 1/1 = гомозигота) |

#### Типы вариантов в VCF

- **SNP:** REF и ALT — одиночные нуклеотиды
- **Индел:** REF длиннее ALT (делеция) или короче (инсерция)
- **Замена блока:** например, REF=ACG, ALT=A (делеция CG)

### 5.5 Фильтрация вариантов

```bash
gatk VariantFiltration \
    -R reference.fa \
    -V sample_variants.vcf \
    -O sample_filtered.vcf \
    --filter-name "QD2" --filter-expression "QD < 2.0" \
    --filter-name "FS60" --filter-expression "FS > 60.0" \
    --filter-name "MQ40" --filter-expression "MQ < 40.0"
```

## Часть 6: Аннотация вариантов

### 6.1 Зачем аннотация?

VCF говорит "где" и "что" изменилось, но не "почему" биологически важно. Нужна аннотация:
- В каком гене
- Влияет ли на белок (missense, synonymous, stop gain)
- Известен ли вариант

### 6.2 Инструменты

| Инструмент | Описание |
|-----------|---------|
| **SnpEff** | Быстрый, простой |
| **ANNOVAR** | Популярный |
| **VEP** | Variant Effect Predictor (Ensembl) |

### 6.3 Пример с SnpEff

```bash
# Скачать базу для GRCh38
snpEff download GRCh38.86

# Аннотировать
snpEff GRCh38.86 sample_filtered.vcf > sample_annotated.vcf
```

### 6.4 Интерпретация аннотации

SnpEff добавляет поле ANN с информацией:
- **Allele** — альтернативный аллель
- **Effect** — эффект (missense_variant, synonymous_variant, stop_gained)
- **Impact** — степень влияния (HIGH, MODERATE, LOW, MODIFIER)
- **Gene Name** — имя гена
- **Gene ID** — ENSEMBL ID

## Часть 7: Популяционная генетика и GWAS

### 7.1 GWAS (Genome-Wide Association Study)

**Цель:** найти статистическую связь между генетическими вариантами и фенотипом

**Метод:** сравнение частот аллелей в группах "случаи" и "контроль"

### 7.2 Множественное тестирование

В GWAS проверяются миллионы SNPs → строгий порог значимости:
- **Бонферрони:** p < 5×10⁻⁸

### 7.3 Тест на ассоциацию (χ²)

**Пример для SNP rs11209026 (IL23R):**

```
                Allele A    Allele G    Всего
Cases           22          976         998
Controls        68          932         1000

χ² ≈ 24.5
p-value ≈ 7.3×10⁻⁷
```

### 7.4 Визуализация GWAS

| График | Описание |
|--------|---------|
| **Manhattan plot** | По оси X хромосомы, по Y −log10(p-value). Пики выше порога = значимые |
| **QQ-plot** | Наблюдаемые vs ожидаемые p-value. Отклонение = сигнал |

### 7.5 Пример: болезнь Крона

Ассоциированные гены: IL23R, ATG16L1, NOD2
→ Помогло понять связь с иммунитетом и аутофагией

### 7.6 Проблемы GWAS

- Большинство ассоциированных вариантов в некодирующих областях
- Часто непонятен целевой ген
- Требуются огромные выборки (сотни тысяч)

## Часть 8: Клиническая интерпретация

### 8.1 Типы генетических тестов

| Тест | Размер | Стоимость | Варианты |
|------|--------|----------|----------|
| **WGS** | 3.1 Гб | ~$1000 | Все включая некодирующие |
| **WES** | 1.5% | ~$300-500 | Только экзоны |
| **Панели** | Сотни генов | ~$100-200 | Заболевание-специфичные |

### 8.2 Аутосомно-рецессивные (AR) заболевания

- Проявляются при повреждении **обеих копий**
- Носители (гетерозиготы) не болеют
- Если оба родителя — носители: **25% риск** больного ребёнка

#### Частые AR заболевания в России

| Заболевание | Ген | Частота | Риск пары |
|-----------|-----|--------|----------|
| Фенилкетонурия | PAH | 2.3% | 1:1890 |
| Муковисцидоз | CFTR | 2.8% | 1:1275 |
| Потеря слуха | GJB2 | 6.8% | 1:216 |

### 8.3 Классификация вариантов (ACMG)

| Класс | Описание | Сообщают пациенту? |
|-------|---------|-------------------|
| **B** (Benign) | Доброкачественный | ❌ |
| **LB** (Likely Benign) | Вероятно доброкачественный | ❌ |
| **VUS** | Неясного значения | ⚠️ Иногда |
| **LP** (Likely Pathogenic) | Вероятно патогенный | ✅ |
| **P** (Pathogenic) | Патогенный | ✅ |

## Контрольные вопросы

**1. Разница между сборкой de novo и ресеквенированием?**

**Ответ:** de novo для новых организмов (сложно, требует ресурсов). Ресеквенирование с известным референсом (быстро, просто).

**2. На каком алгоритме основан BWA?**

**Ответ:** Преобразование Барроуза-Уилера (BWT) и FM-индекс.

**3. Что такое BWT и зачем?**

**Ответ:** BWT переупорядочивает строку для эффективного сжатия и поиска. Позволяет быстро находить совпадения в больших геномах.

**4. Поля SAM-файла?**

**Ответ:** QNAME, FLAG, RNAME, POS, MAPQ, CIGAR, RNEXT, PNEXT, TLEN, SEQ, QUAL.

**5. Как конвертировать SAM в BAM и зачем?**

**Ответ:** `samtools sort` → бинарный формат, меньше памяти, быстрее обработка, требуется для downstream.

**6. ПЦР-дубликаты и удаление?**

**Ответ:** Риды от одной молекулы → искажают частоты. Удаляют через GATK MarkDuplicates.

**7. Программы для поиска вариантов?**

**Ответ:** GATK HaplotypeCaller, FreeBayes, bcftools, DeepVariant, Strelka2.

**8. VCF-файл?**

**Ответ:** Текстовый формат с вариантами. Колонки: CHROM, POS, ID, REF, ALT, QUAL, FILTER, INFO, FORMAT, sample.

**9. SNP, индел, SV?**

**Ответ:** SNP — замена одного нуклеотида. Индел — короткие вставки/делеции. SV — крупные перестройки.

**10. Аннотация вариантов?**

**Ответ:** SnpEff, ANNOVAR, VEP определяют ген, эффект (missense/synonymous), влияние на белок.

**11. GWAS и Манхэттенский график?**

**Ответ:** GWAS ищет ассоциации вариантов с фенотипом. Манхэттен: хромосомы на X, −log10(p-value) на Y, пики = значимые.

**12. Порог значимости 5×10⁻⁸?**

**Ответ:** Коррекция Бонферрони за ~1 млн тестов в GWAS.

**13. AR заболевание и риск рождения больного?**

**Ответ:** Проявляется при повреждении обеих копий. Оба родителя носители → **25% риск**.

**14. Классы патогенности (B, LB, VUS, LP, P)?**

**Ответ:** B/LB — безопасны, не сообщают. VUS — неясны, иногда сообщают. LP/P — патогенны, сообщают.
