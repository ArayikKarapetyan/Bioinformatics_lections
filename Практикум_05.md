## Часть 1: Теоретические основы BLAST

### 1.1 Задача, которую решает BLAST

**BLAST** (Basic Local Alignment Search Tool) — это алгоритм для быстрого поиска гомологичных последовательностей в большой базе данных.

Он ищет **локальные выравнивания** между запросом (query) и каждой последовательностью в базе данных (subject).

#### Основные применения (как в лекции):

1. **Проверка наличия** — есть ли моя последовательность в базе данных
2. **Поиск аннотированных гомологов** — для предсказания функции
3. **Поиск известных структур** — гомологи с известной пространственной структурой
4. **Поиск в новых геномах** — белки с интересующей функцией в новом геноме/транскриптоме

### 1.2 Почему не подходит Smith-Waterman?

**Smith-Waterman** даёт оптимальное локальное выравнивание, но его **сложность O(n×m) слишком велика** для поиска по огромным базам данных (миллиарды оснований).

**BLAST использует эвристики** для ускорения в сотни раз, жертвуя небольшой чувствительностью.

### 1.3 Три этапа алгоритма BLAST

#### Этап 1: Индексация базы данных

1. **База данных разбивается на слова** (k-mer'ы) фиксированной длины k
   - Для нуклеотидов по умолчанию: **k = 28** (megaBLAST) или **k = 11** (blastn)
   - Для белков по умолчанию: **k = 3** (или 2 в некоторых вариантах)

2. **Кодирование:** каждому k-mer'у присваивается числовой код
   - Пример: A=0, C=1, G=2, T=3, тогда TGAC = 3201₄ = 225₁₀

3. **Индекс:** составляется отсортированный список всех k-mer'ов с указанием позиций в базе
   - Позволяет искать любой k-mer за O(log N) с помощью бинарного поиска

#### Этап 2: Поиск seed-ов (затравок)

1. **Запрос разбивается на k-mer'ы**

2. **Для каждого k-mer'а запроса** генерируются все похожие k-mer'ы, чей скор выше порога T
   - Для белков T обычно = 11-13 при использовании BLOSUM62

3. **Все похожие k-mer'ы ищутся в индексе** базы данных
   - Найденные позиции называются **HSP** (High-scoring Segment Pairs) начального уровня

#### Этап 3: Удлинение (extension)

1. **Каждый найденный seed расширяется в обе стороны** без гэпов (ungapped extension)
   - Пока скор падает ниже порога X

2. **Гэппированное выравнивание** (gapped extension)
   - Запускается, если расширенный HSP имеет скор выше порога S
   - Использует алгоритм, похожий на Smith-Waterman, но в окне вокруг seed

3. **Статистическая значимость** вычисляется для каждого значимого HSP (E-value)

### 1.4 Влияние параметра k (word size)

| Параметр | Маленький k | Большой k |
|----------|------------|-----------|
| Чувствительность | Выше (находятся отдаленные гомологи) | Ниже (пропускаются отдаленные) |
| Скорость | Медленнее | Быстрее |
| Использование памяти | Больше | Меньше |
| Лучше для | Поиск далеких гомологов | Поиск идентичных последовательностей |

#### Значения по умолчанию в NCBI BLAST:

- **Нуклеотиды (megablast):** 28
- **Нуклеотиды (blastn):** 11
- **Белки (blastp):** 3
- **PSI-BLAST:** 2

### 1.5 Статистическая значимость: Bit score и E-value

#### Bit score (S')

**Нормализованный скор**, не зависящий от матрицы замен и размера базы данных:

```
S' = (λ·S - ln K) / ln 2
```

где λ и K — параметры Карлина–Альтшуля, зависящие от матрицы замен и частот символов.

#### E-value (Expectation value)

**Ожидаемое число случайных выравниваний** с таким же или лучшим скором:

```
E = K·m·n·e^{-λ·S}
```

или в терминах bit score:

```
E = m·n·2^{-S'}
```

где:
- m = длина запроса
- n = общая длина базы данных (сумма длин всех последовательностей)

#### P-value

**Вероятность получить хотя бы одно такое выравнивание случайно:**

```
P = 1 - e^{-E}
```

При малых E (<<1) P ≈ E.

#### Эмпирические правила для белков (из лекции):

| E-value | Интерпретация |
|---------|---------------|
| E < 1e-100 | Идентичные последовательности |
| 1e-50 < E < 1e-100 | Почти идентичные |
| 1e-5 < E < 1e-50 | Гомологичные |
| 1e-1 < E < 1e-5 | Отдаленные гомологи |
| E > 1e-1 | Возможно, случайные совпадения |

## Часть 2: Работа с веб-версией BLAST на NCBI

### 2.1 Знакомство с интерфейсом

Зайдите на главную страницу **[NCBI BLAST](https://blast.ncbi.nlm.nih.gov)**

Вы увидите несколько вариантов:

| Тип | Назначение |
|-----|-----------|
| **Nucleotide BLAST (blastn)** | Для нуклеотидных запросов |
| **Protein BLAST (blastp)** | Для белковых запросов |
| **blastx** | Нуклеотидный запрос против белковой базы (транслируется на лету) |
| **tblastn** | Белковый запрос против нуклеотидной базы (транслируется) |
| **tblastx** | Нуклеотидный запрос против нуклеотидной базы (оба транслируются) |

### 2.2 Варианты нуклеотидного BLAST (из лекции)

| Вариант | Использование | Word size | Скорость |
|---------|--------------|-----------|----------|
| **Megablast** | Очень похожие (>95% идентичности) | 28 | Очень быстро |
| **Discontiguous megablast** | Межвидовые сравнения | переменный | Быстро |
| **blastn** | Классический, чувствительный | 11 | Медленнее |

### 2.3 Варианты белкового BLAST (из лекции)

| Вариант | Назначение |
|---------|-----------|
| **Quick BLASTP** | Ускоренная версия для проверки наличия в базе |
| **blastp** | Стандартный поиск |
| **PSI-BLAST** | Итеративный поиск для отдаленных гомологов; на каждой итерации строит PSSM |
| **PHI-BLAST** | Поиск по паттерну (мотиву) |
| **DELTA-BLAST** | Поиск через консервативные домены (CDD); очень чувствительный |

### 2.4 Практическое задание: поиск гемоглобина у приматов

#### Цель:
Найти гомологи бета-глобина человека у других приматов, исключив самого человека.

#### Шаги:

1. **Получение запроса:**
   - На NCBI найдите запись бета-глобина человека, например, accession number **DQ659148.1**
   - Скопируйте этот номер (BLAST сам возьмёт последовательность)

2. **Переход к BLAST:**
   - Нажмите "Nucleotide BLAST"
   - Вставьте DQ659148.1 в поле ввода

3. **Выбор базы данных:**
   - Оставьте стандартную базу "nt" (нуклеотидная коллекция)

4. **Ограничение по организму:**
   - В разделе "Organism" введите "Primates" (приматы)
   - Нажмите "Add organism"
   - Поставьте галочку "exclude" и введите "Homo sapiens", чтобы исключить человека

5. **Выбор алгоритма:**
   - Попробуйте разные варианты:
     - **Megablast** (word size 28)
     - **Discontiguous megablast**
     - **blastn** (word size 11)

#### Сравнение результатов:

- Сколько находок дал каждый алгоритм?
- Какие E-value у лучших находок?
- Есть ли разница в покрытии запроса?

### 2.5 Анализ результатов BLAST

После поиска вы увидите таблицу результатов с колонками:

| Колонка | Описание |
|---------|---------|

| **Per. Ident** | Процент идентичности |
| **Accession** | Номер в базе данных |

#### Задание:

Для лучших 5 находок запишите:
- Организм

- % идентичности
- Длину выравнивания

## Часть 3: Белковый BLAST — Поиск гомологов Лингулы-1 (LY6/UPAR)

### 3.1 Поиск гомологов у иглокожих (морские звёзды)

**LY6/UPAR семейство** — белки, модулирующие никотиновые холиновые рецепторы.

#### Шаги:

1. **Запрос:**
   - Возьмём человеческий белок LY6/UPAR, например, с UniProt ID **Q9Y5K5** (Lynx1)
   - Скопируйте его последовательность или используйте accession в BLAST

2. **Переход:**
   - Выберите "Protein BLAST"

3. **База данных:**
   - Оставьте "nr" (non-redundant protein sequences)

4. **Организм:**
   - Введите "Echinodermata" (иглокожие — морские звёзды, ежи и др.)
   - Можно исключить человека

5. **Алгоритм:**
   - Выберите **blastp** (стандартный)

6. **Параметры:**
   - **Max target sequences:** 500
   - **Expect threshold:** 0.05 (по умолчанию)
   - **Word size:** 3
   - **Matrix:** BLOSUM62
   - **Gap costs:** Existence 11, Extension 1

#### Вопросы для анализа:

- Какие E-value у лучших находок?
- Есть ли находки из рода Asterias (морские звёзды)?
- Какой процент идентичности у самых далёких гомологов?

### 3.2 Использование PSI-BLAST для глубокого поиска

**PSI-BLAST** позволяет найти очень отдаленные гомологи, строя профиль (PSSM) из найденных последовательностей.

#### Шаги:

1. **На странице результатов blastp** нажмите кнопку "Run PSI-BLAST"

2. **Первая итерация:** по сути повтор blastp
   - После завершения увидите список находок
   - Некоторые будут отмечены как "New" (новые для этой итерации)

3. **Выбор последовательностей:**
   - Выберите галочками те последовательности, которые считаете достоверными гомологами
   - Обычно все с E-value < 0.005

4. **Запуск итерации 2:**
   - Нажмите "Run PSI-BLAST iteration 2"
   - Запросом будет уже PSSM, построенный по выбранным последовательностям

5. **Повторение:**
   - Повторите 3-5 итераций
   - Заметьте, как растёт количество находок и меняются E-value

#### Вопрос:

**Почему PSI-BLAST находит больше гомологов, чем обычный blastp?**

**Ответ:** Потому что профиль (PSSM) содержит информацию об эволюционных замах на каждой позиции, полученную из выравнивания найденных на предыдущих итерациях гомологов.

### 3.3 Использование DELTA-BLAST

1. **Вернитесь к странице запуска** белкового BLAST

2. **В разделе "Algorithm"** выберите **DELTA-BLAST**

3. **Запустите поиск** с тем же запросом и ограничением на иглокожих

4. **Сравните результаты:**
   - Есть ли различия в первых находках?
   - Отличаются ли E-value?
   - Как DELTA-BLAST использует консервативные домены?

## Часть 4: Трансляемые BLAST (blastx, tblastn, tblastx)

### 4.1 Зачем нужна трансляция?

**Нуклеотидные последовательности** могут кодировать белки в **одной из шести рамок считывания**:
- 3 прямой цепи
- 3 обратной комплементарной

**Трансляционные BLAST** позволяют сравнивать на белковом уровне, что **чувствительнее** из-за эволюционных ограничений на аминокислоты.

### 4.2 blastx: нуклеотидный запрос против белковой базы

#### Сценарий:
У вас есть кДНК или геномный фрагмент, и вы хотите узнать, какой белок он кодирует.

#### Шаги:

1. **На главной странице BLAST** выберите **blastx**

2. **Вставьте нуклеотидную последовательность** (например, часть кДНК гемоглобина)

3. **Оставьте базу "nr"** и ограничьте организмом, например, "Mammalia"

4. **Запустите поиск**

#### Анализ результатов:

- Выдача будет содержать белковые последовательности
- Будет указано, в какой рамке считывания произошло выравнивание (Frame +1, +2, +3, -1, -2, -3)

### 4.3 tblastn: белковый запрос против нуклеотидной базы

#### Сценарий:
У вас есть белковая последовательность, и вы хотите найти кодирующий её ген в неаннотированном геноме.

#### Шаги:

1. **Выберите tblastn**

2. **Вставьте белковую последовательность** (например, Lynx1)

3. **В качестве базы данных** выберите, например, "Whole-genome shotgun contigs" (WGS)

4. **Запустите поиск**

#### Результаты:

- Покажут участки генома, которые при трансляции дают похожую аминокислотную последовательность

### 4.4 tblastx: нуклеотид против нуклеотида с трансляцией обоих

**Используется реже**, обычно для сравнения двух **неаннотированных нуклеотидных последовательностей** на белковом уровне.

#### Когда использовать tblastx:

- Если у вас есть две нуклеотидные последовательности
- Вы подозреваете, что они кодируют похожие белки
- Нуклеотидные последовательности сильно разошлись

## Часть 5: Запуск BLAST из командной строки (локально)

### 5.1 Подготовка данных

На сервере (или локально) создайте папку и файлы:

```bash
mkdir blast_demo
cd blast_demo
```

Вам нужны два файла:

1. **База данных:** `arubens_protein.fa` (белковые последовательности из генома Asterias rubens)
   - Если нет реального файла, скачайте с NCBI или создайте искусственный

2. **Запрос:** `ly6_upar.fasta` (несколько известных LY6-белков)

#### Пример файла запроса:

```fasta
>sp|Q9Y5K5|LYNX1_HUMAN
MKKMLLLILFLVLVVPSQGQLEKCYVCKEPLAPKDELDETVCSDGNIYCNGVLKIH
>sp|P0DP58|LYNX1_MOUSE
MKKMLLLILFLVLVVPSQGQLEKCYVCKEPLAPKDELDETVCSDGNIYCNGVLKIH
>sp|P0DP57|LYNX1_RAT
MKKMLLLILFLVLVVPSQGQLEKCYVCKEPLAPKDELDETVCSDGNIYCNGVLKIH
>sp|Q9PS82|BML1_NAJAT
MKTLLLTLVVVTIVCLDLGYTRGCHECNQCSSEPENTTCTTSGKICYKKFWKCT
```

### 5.2 Индексация базы данных (makeblastdb)

Перед поиском базу данных нужно проиндексировать:

```bash
makeblastdb -in arubens_protein.fa -dbtype prot -out arubens_protein
```

**Параметры:**

| Параметр | Описание |
|----------|---------|
| `-dbtype prot` | Тип базы: белки (prot) или нуклеотиды (nucl) |
| `-out` | Префикс выходных файлов (.phr, .pin, .psq) |

**Проверьте результат:**

```bash
ls arubens_protein*
```

Должны появиться файлы: `.phr`, `.pin`, `.psq`

### 5.3 Запуск blastp

```bash
blastp -query ly6_upar.fasta -db arubens_protein -out results.txt -outfmt 6
```

**Параметры:**

| Параметр | Описание |
|----------|---------|
| `-query` | Файл с запросом (несколько последовательностей) |
| `-db` | Имя базы данных (префикс, без расширений) |
| `-out` | Файл для результатов |
| `-outfmt 6` | Табличный формат (tab-separated), удобный для анализа |

**Другие популярные форматы:**

- `-outfmt 0` — парное выравнивание с описаниями (как на сайте)
- `-outfmt 7` — табличный с комментариями
- `-outfmt 10` — CSV

### 5.4 Просмотр результатов

```bash
cat results.txt | head -20
```

#### Столбцы в формате 6 (по умолчанию):

| # | Название | Описание |
|---|----------|---------|
| 1 | qseqid | идентификатор запроса |
| 2 | sseqid | идентификатор найденной последовательности |
| 3 | pident | процент идентичности |
| 4 | length | длина выравнивания |
| 5 | mismatch | количество несовпадений |
| 6 | gapopen | количество открытых гэпов |
| 7 | qstart | старт на запросе |
| 8 | qend | конец на запросе |
| 9 | sstart | старт на найденной |
| 10 | send | конец на найденной |
| 11 | evalue | E-value |
| 12 | bitscore | bit score |

### 5.5 Фильтрация результатов

Найдём только те выравнивания, у которых E-value < 1e-5:

```bash
awk '$11 < 1e-5' results.txt
```

### 5.6 Запуск с другими параметрами

Попробуйте изменить word size и матрицу замен:

```bash
blastp -query ly6_upar.fasta -db arubens_protein \
```

## Контрольные вопросы (расширенные)

### Теоретические вопросы

**1. Какова вычислительная сложность алгоритма Смита–Ватермана и BLAST? Почему BLAST так быстрее?**

**Ответ:** Smith-Waterman имеет сложность O(n×m), где n и m — длины последовательностей. BLAST использует индексирование (O(log N) для поиска k-мера) и работает с seed'ами, что даёт сложность близкую к линейной O(n+m) в среднем случае. Это дает ускорение в сотни-тысячи раз.

**2. Что такое k-mer, word size? Как его выбор влияет на чувствительность и скорость поиска?**

**Ответ:** k-mer — слово длины k. Маленький k => больше seed'ов => выше чувствительность, но медленнее. Большой k => мало seed'ов => быстрее, но пропускаются отдаленные гомологи.

**3. Зачем нужна индексация базы данных? Как она реализована в BLAST?**

**Ответ:** Индексация позволяет быстро (за O(log N)) найти позиции всех k-мер'ов в базе. Реализация: отсортированный список всех k-мер'ов с указателями на позиции в базе.

**4. Что такое HSP (high-scoring segment pair) и как он получается из seed?**

**Ответ:** HSP — выравнивание без гэпов между запросом и последовательностью базы. Получается путём расширения seed в обе стороны, пока скор выше порога.

**5. Объясните разницу между E-value и P-value. Почему для оценки значимости используют E-value?**

**Ответ:** E-value — ожидаемое число случайных выравниваний с таким же скором; P-value — вероятность получить хотя бы одно. E-value используют, потому что это зависит от размера базы (большая база => больше случайных совпадений).

**6. Что такое bit score и почему он лучше сырого скора для сравнения результатов из разных поисков?**

**Ответ:** Bit score — нормализованный скор, не зависящий от матрицы замен и размера базы. Это позволяет сравнивать результаты разных поисков напрямую.

### Практические вопросы

**7. В каких случаях стоит использовать PSI-BLAST вместо обычного blastp?**

**Ответ:** Когда нужно найти отдаленные гомологи (E > 1e-5). PSI-BLAST строит профиль (PSSM) из найденных последовательностей, что повышает чувствительность на итерациях.

**8. Какие существуют варианты нуклеотидного BLAST и для каких задач каждый из них предназначен?**

**Ответ:**
- **Megablast** — для очень похожих (>95%), быстро
- **Discontiguous** — для межвидовых сравнений
- **blastn** — универсальный, чувствительный

**9. Что такое открытая рамка считывания (ORF) и как её найти?**

**Ответ:** ORF — участок ДНК от старт-кодона (ATG) до стоп-кодона (TAA, TAG, TGA). Находится путём сканирования всех трёх рамок считывания на обеих цепях.

**10. Какой трансляционный BLAST вы выберете, если:**

**a) у вас есть кДНК и вы хотите найти похожие белки в другом виде?**
→ **blastx**

**b) у вас есть белок и вы хотите найти его ген в неаннотированном геноме?**
→ **tblastn**

**c) у вас есть две нуклеотидные последовательности, и вы подозреваете, что они кодируют похожие белки, но нуклеотидные последовательности сильно разошлись?**
→ **tblastx**
