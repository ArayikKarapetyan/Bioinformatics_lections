## Часть 1: Создание SSH-ключей

SSH-ключи нужны для безопасного подключения к серверу. Мы создадим пару ключей:
- **Закрытый (приватный)** — хранится только у вас
- **Открытый (публичный)** — его нужно будет отправить администратору сервера

### Если у вас Windows (с помощью Bitvise)

#### Пошаговое руководство

1. **Запустите** установленный Bitvise SSH Client

2. **В главном окне** нажмите кнопку **Client key manager** (менеджер ключей)

3. **В открывшемся окне** нажмите **Generate New** (сгенерировать новый)

4. **В настройках генерации:**
   - **Algorithm (Алгоритм):** выберите **RSA**
   - **Size (Размер):** установите **4096 бит** (рекомендация из лекции)

5. **Нажмите Generate**

6. **Дождитесь завершения генерации.** Новый ключ появится в списке

7. **Выделите его** и нажмите **Export**

8. **В появившемся окне:**
   - Обязательно поставьте галочку **"OpenSSH format"** (формат OpenSSH)
   - Это стандартный формат, который понимает сервер

9. **Нажмите Export.** Сохраните публичный ключ
   - Рекомендуется создать папку `ssh` в ваших документах (например, `C:\Users\ВашеИмя\Documents\ssh\`)
   - Назовите файл `id_rsa.pub`

10. **Откройте сохраненный файл** `id_rsa.pub` с помощью Блокнота
    - Вы увидите длинную строку, начинающуюся с `ssh-rsa AAAA....`
    - Скопируйте её полностью в буфер обмена
    - **Это ваш публичный ключ**

### Если у вас MacOS или Linux

#### Пошаговое руководство

1. **Откройте Терминал**
   - **MacOS:** Finder → Программы → Утилиты → Терминал, или через Spotlight (Cmd+Пробел, введите "Терминал")
   - **Linux:** обычно есть в меню приложений или вызывается сочетанием клавиш (Ctrl+Alt+T)

2. **Убедитесь, что вы в домашней директории**
   - Просто откройте терминал, вы будете там
   - Для верности можно выполнить команду `cd` (без аргументов) — она всегда возвращает в домашнюю папку

3. **Введите команду для генерации ключа:**

```bash
ssh-keygen -b 4096
```

**Расшифровка:**
- `ssh-keygen` — программа для создания ключей
- `-b 4096` — флаг, указывающий размер ключа 4096 бит

4. **Система спросит, куда сохранить ключ:**

```
Enter file in which to save the key (/Users/ваше_имя/.ssh/id_rsa):
```

Просто нажмите **Enter**, чтобы согласиться с путём по умолчанию (`~/.ssh/id_rsa`)

5. **Затем система попросит ввести парольную фразу (passphrase)**

Это дополнительная защита вашего приватного ключа. Если кто-то получит ваш файл ключа, без пароля он не сможет им воспользоваться.

**Варианты:**
- Можно ввести пароль (он не будет отображаться при вводе)
- Можно оставить пустым, просто нажав Enter

⚠️ **Важно:** если вы установите пароль, его нужно будет вводить при каждом подключении к серверу

6. **Подтвердите пароль** (или ещё раз нажмите Enter, если оставили пустым)

7. **Ключи созданы!** Теперь нужно скопировать содержимое публичного ключа

Выполните команду:

```bash
cat ~/.ssh/id_rsa.pub
```

На экране отобразится содержимое файла (строка, начинающаяся с `ssh-rsa AAA...`).

8. **Выделите её мышью** и скопируйте (обычно Ctrl+Shift+C в терминале Linux, Cmd+C в MacOS)

## Часть 2: Отправка публичного ключа администратору (имитация)

В реальной ситуации вы отправляете скопированный ключ (и свои данные) администратору.

В рамках этого практикума мы просто сохраним его в текстовый файл на рабочем столе с именем `my_public_key.txt`. Это будет нашей "отправкой".

#### Инструкции

1. **Откройте Блокнот** (или любой текстовый редактор)

2. **Вставьте скопированный ключ**

3. **Сохраните файл** на рабочем столе под именем `my_public_key.txt`

## Часть 3: Подключение к серверу

Теперь, когда администратор создал для вас аккаунт (в реальности), вы можете подключиться.

### У вас должны быть:

- **Имя пользователя (username)** — которое вам выдал администратор
- **Адрес сервера (host)** — например, `frontend.aldan3.itm-rsmu.ru`
- **Пароль** (возможно, временный) или доступ по ключу

⚠️ **Примечание:** В этом задании мы будем подключаться к серверу, доступ к которому вам предоставят преподаватели. Если сервер ещё не готов, вы можете потренироваться подключаться к локальной виртуальной машине или использовать WSL, но лучше дождаться доступа к реальному серверу.

### Подключение с Windows (Bitvise)

1. **Запустите Bitvise SSH Client**

2. **В поле Host** введите адрес сервера (например, `frontend.aldan3.itm-rsmu.ru`)

3. **Port** оставьте **22** (стандартный порт SSH)

4. **Username** введите ваше имя пользователя, которое вам сообщили

5. **В поле Initial method** выберите **publickey** (аутентификация по ключу)

6. **В выпадающем списке Client key** выберите тот ключ, который мы создавали в Части 1
   - Он должен там появиться автоматически, если вы его экспортировали в менеджере ключей
   - Если его нет, нажмите "Client key manager" и импортируйте ваш приватный ключ (файл без расширения или с расширением `.ppk`)

7. **Нажмите кнопку Log in**

8. **При первом подключении** появится окно с вопросом, доверяете ли вы этому серверу
   - Это окно показывает публичный ключ сервера
   - Нужно нажать **"Accept and Save"** (Принять и сохранить)
   - Теперь ваш компьютер знает этот сервер и будет доверять ему при следующих подключениях

9. **Если всё сделано правильно**, вы увидите:
   - Сообщения об успешной аутентификации
   - Откроется терминал сервера
   - Также появится возможность открыть SFTP-окно для обмена файлами (кнопка "New SFTP window")

### Подключение с MacOS / Linux

1. **Откройте Терминал**

2. **Выполните команду:**

```bash
ssh ваше_имя_пользователя@адрес_сервера
```

**Пример:**

```bash
ssh ivanov@frontend.aldan3.itm-rsmu.ru
```

3. **При первом подключении** вы увидите сообщение:

```
The authenticity of host '...' can't be established.
RSA key fingerprint is ...
Are you sure you want to continue connecting (yes/no)?
```

Это аналог окна с вопросом о доверии серверу.

4. **Введите yes** и нажмите **Enter**
   - Ключ сервера будет сохранён
   - Вы больше не увидите это сообщение для данного сервера

5. **Если для вашего ключа была установлена парольная фраза**, система попросит ввести её
   - Если нет, подключение произойдёт автоматически

6. **Вы увидите:**
   - Приветственное сообщение
   - Приглашение командной строки сервера (обычно заканчивается на `$`)

🎉 **Поздравляю, вы на сервере!**

## Часть 4: Основы навигации и работы с файлами в Linux

Теперь, когда мы подключены к серверу, давайте изучим основные команды. Вводите команды и смотрите на результат.

### 4.1 Ориентация в пространстве

#### pwd (Print Working Directory)

Показать, в какой папке мы сейчас находимся.

```bash
pwd
```

Вы должны увидеть что-то вроде `/home/ваше_имя`. Это ваша домашняя директория.

#### ls (List)

Показать содержимое текущей папки.

```bash
ls
```

Скорее всего, пока там пусто.

**Попробуйте разные флаги:**

```bash
ls -l        # показать в длинном формате (права доступа, владелец, размер, дата)
ls -a        # показать все файлы, включая скрытые (их имена начинаются с точки)
ls -la       # комбинация двух флагов
```

### 4.2 Перемещение по папкам

#### cd (Change Directory)

Сменить директорию.

**Варианты использования:**

```bash
cd имя_папки    # Перейти в конкретную папку
cd ..           # Подняться на уровень выше
cd ~            # Перейти в домашнюю директорию
cd              # Перейти в домашнюю директорию (без аргументов)
cd /            # Перейти в корневую директорию
```

#### Задание 1: Путешествие по файловой системе

1. **Перейдите в корневую директорию:**

```bash
cd /
```

2. **Выполните ls** и посмотрите, какие там есть папки
   - Найдите среди них `home`, `etc`, `var`, `tmp`
   - Сравните со слайдом "Структура папок в Linux"

3. **Перейдите в папку home:**

```bash
cd home
```

4. **Перейдите в свою домашнюю папку:**

```bash
cd ваше_имя
```

5. **Убедитесь, что вы дома:**

```bash
pwd
```

### 4.3 Создание и удаление папок и файлов

#### mkdir (Make Directory)

Создать папку.

```bash
mkdir test_dir
```

**Проверьте ls**, появилась ли папка `test_dir`.

#### touch

Создать пустой файл или обновить время модификации существующего.

```bash
touch file1.txt
touch file2.txt
```

**Проверьте ls.**

#### cp (Copy)

Копировать файлы или папки.

```bash
cp file1.txt file1_copy.txt                # копировать файл
cp -r test_dir test_dir_copy               # копировать папку рекурсивно
```

#### mv (Move)

Переместить или переименовать.

```bash
mv file1.txt new_file_name.txt             # переименовать файл
mv new_file_name.txt test_dir/             # переместить файл в другую папку
```

#### rm (Remove)

Удалить.

```bash
rm file2.txt                               # удалить файл
rm -d test_dir_copy                        # удалить пустую папку (или rmdir)
rm -rf test_dir                            # удалить папку со всем содержимым рекурсивно и принудительно
```

⚠️ **БУДЬТЕ ОСТОРОЖНЫ!** Последняя команда удаляет всё необратимо.

### 4.4 Просмотр содержимого файлов

#### Создание тестового файла

Создадим файл с текстом для упражнений. Напечатайте несколько строк в файл `poem.txt`:

```bash
echo "Вот море, вот лес, вот луг прекрасный." > poem.txt
echo "А это — тропинка, ведущая в сад." >> poem.txt
echo "А это — овечка, которая спит" >> poem.txt
echo "У тропинки, ведущей в сад." >> poem.txt
```

**Разберём команды:**

| Команда | Описание |
|---------|---------|
| `echo "текст"` | Выводит текст в терминал |
| `>` | Перенаправляет вывод команды в файл, создавая его или перезаписывая |
| `>>` | Перенаправляет вывод команды в файл, дописывая в конец |

#### Просмотр файлов

##### cat (Concatenate)

Вывести всё содержимое файла в терминал.

```bash
cat poem.txt
```

##### less

Просмотр файла постранично (удобно для больших файлов).

```bash
less poem.txt
```

**Навигация:** стрелки вверх/вниз, пробел — страница вниз, `q` — выход.

##### head

Посмотреть первые строки файла (по умолчанию 10).

```bash
head -n 2 poem.txt        # показать первые 2 строки
```

##### tail

Посмотреть последние строки файла.

```bash
tail -n 1 poem.txt        # показать последнюю строку
```

##### wc (Word Count)

Посчитать строки, слова, символы.

```bash
wc poem.txt               # показать всё
wc -l poem.txt            # только строки
wc -w poem.txt            # только слова
```

### 4.5 Полезные команды

#### history

Показать историю введённых команд.

```bash
history
```

#### clear

Очистить экран терминала.

```bash
clear
```

#### tree

Показать структуру папок в виде дерева (Может быть не установлена, тогда пропустите).

```bash
tree ~/                   # показать дерево домашней папки
```

#### du (Disk Usage)

Оценить размер файлов и папок.

```bash
du -sh ~/                 # суммарный размер домашней папки в человеко-читаемом формате
```

### 4.6 Конвейеры (Pipe) — мощный инструмент

Символ `|` (вертикальная черта) передаёт результат выполнения одной команды на вход другой.

#### Задание 3: Конвейеры

##### Посчитайте количество строк содержащих слово

Посчитайте, сколько строк в файле `poem.txt` содержит слово "тропинка".

Команда `grep` ищет текст в файлах. Попробуйте так:

```bash
grep "тропинка" poem.txt | wc -l
```

**Что произошло?**
- `grep` нашёл строки со словом "тропинка"
- Передал их команде `wc -l`
- Та посчитала количество строк

##### Сохраните строки в новый файл

Посмотрите последние 3 строки файла `poem.txt` и сохраните их в новый файл `end_poem.txt`.

```bash
tail -n 3 poem.txt > end_poem.txt
```

## Часть 5: Знакомство с текстовым редактором nano

На сервере часто нужно редактировать файлы конфигурации или писать небольшие скрипты прямо в терминале.

**nano** — самый простой редактор для начинающих.

### Создание и редактирование файла

1. **Создайте и откройте файл** `my_first_script.sh` для редактирования:

```bash
nano my_first_script.sh
```

2. **Вы попадёте в редактор.** Напишите внутри (или скопируйте) следующий текст — это будет простой bash-скрипт:

```bash
#!/bin/bash
echo "Привет, мир!"
echo "Я учусь работать в Linux."
pwd
```

3. **Сохраните файл:**
   - Нажмите **Ctrl+O** (WriteOut)
   - Внизу появится запрос на подтверждение имени файла
   - Просто нажмите **Enter**

4. **Выйдите из nano:**
   - Нажмите **Ctrl+X**

### Запуск скрипта

Теперь сделаем скрипт исполняемым и запустим его:

```bash
chmod +x my_first_script.sh       # добавляем право на исполнение
./my_first_script.sh              # запускаем скрипт
```

Вы должны увидеть вывод скрипта.

### Ответы на контрольные вопросы

1. **SSH** (Secure Shell) — это протокол для безопасного подключения к удаленному серверу через зашифрованное соединение

2. **Публичный ключ** отправляется серверу (его знают все), **приватный** хранится только у вас (его никому не показываем)

3. **`pwd`** (print working directory)

4. **`cd ~`** или просто **`cd`** (без аргументов)

5. **`>`** перезаписывает файл, **`>>`** дописывает в конец

6. **`cp -r`** копирует папку со всем содержимым рекурсивно

7. **`head -n 5 имя_файла`**

8. Используя символ **`|`** (pipe), например: `команда1 | команда2`

9. **`chmod +x имя_файла`**

10. **`clear`**
