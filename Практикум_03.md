### 2.1. Функция для расстояния Хэмминга

```python
def hamming(seq1, seq2):
    if len(seq1) != len(seq2):
        raise ValueError("Последовательности разной длины!")
    return sum(a != b for a, b in zip(seq1, seq2))
```

## Часть 3. Алгоритм Нидлмана–Вунша (глобальное выравнивание) — ручной разбор

### 3.1. Условие задачи

Возьмём две короткие последовательности:

```
v = T C G A T T A
w = T C A T T G
```

**Скоринг:** совпадение (match) = +1, несовпадение (mismatch) = -1, гэп (gap) = -1.

### 3.2. Построение матрицы вручную

Нарисуйте на бумаге таблицу размером `(len(v)+1) x (len(w)+1)`. Подпишите строки и столбцы.

**Инициализация первой строки и первого столбца:**
- `(0,0)` = 0
- Каждый шаг вправо: добавляем -1
- Каждый шаг вниз: добавляем -1

**Заполнение по рекуррентной формуле:**

Для каждой ячейки `(i,j)` считаем три значения:

1. **Диагональ:** `score[i-1][j-1] + (match если v[i-1] == w[j-1], иначе mismatch)`
2. **Вверх:** `score[i-1][j] + gap`
3. **Влево:** `score[i][j-1] + gap`

Записываем максимум.

**Backtracking:** начиная с правого нижнего угла, идём обратно к `(0,0)` по стрелкам (откуда пришли). Если пришли по диагонали — обе буквы используются; сверху — гэп во второй последовательности; слева — гэп в первой.

**Задание:** воспроизведите таблицу из лекции. Должно получиться:

```
    T  C  A  T  T  G
    0 -1 -2 -3 -4 -5 -6
 T -1  1  0 -1 -2 -3 -4
 C -2  0  2  1  0 -1 -2
 G -3 -1  1  1  2  1  0
 A -4 -2  0  2  1  1  0
 T -5 -3 -1  1  3  2  1
 T -6 -4 -2  0  2  4  3
 G -7 -5 -3 -1  1  3  3
```

**Оптимальное выравнивание:**

```
TCGATTA
TC-ATTG
```

Скор = 3.

### 3.3. Реализация алгоритма на Python (пошаговая)

Напишем функцию, которая возвращает не только скор, но и все возможные оптимальные выравнивания (их может быть несколько).

```python
def needleman_wunsch_all(seq1, seq2, match=1, mismatch=-1, gap=-1):
    n, m = len(seq1), len(seq2)
    score = np.zeros((n+1, m+1), dtype=int)
    # Для хранения направлений: 0 - диагональ, 1 - вверх, 2 - влево (может быть несколько)
    pointers = [[[] for _ in range(m+1)] for _ in range(n+1)]

    # Инициализация
    for i in range(1, n+1):
        score[i][0] = score[i-1][0] + gap
        pointers[i][0].append(1)  # вверх
    for j in range(1, m+1):
        score[0][j] = score[0][j-1] + gap
        pointers[0][j].append(2)  # влево

    # Заполнение
    for i in range(1, n+1):
        for j in range(1, m+1):
            diag = score[i-1][j-1] + (match if seq1[i-1] == seq2[j-1] else mismatch)
            up = score[i-1][j] + gap
            left = score[i][j-1] + gap
            max_score = max(diag, up, left)
            score[i][j] = max_score
            if diag == max_score:
                pointers[i][j].append(0)
            if up == max_score:
                pointers[i][j].append(1)
            if left == max_score:
                pointers[i][j].append(2)

    # Рекурсивный сбор всех выравниваний
    def backtrack(i, j):
        if i == 0 and j == 0:
            return [("", "")]
        results = []
        for p in pointers[i][j]:
            if p == 0:  # диагональ
                for a1, a2 in backtrack(i-1, j-1):
                    results.append((a1 + seq1[i-1], a2 + seq2[j-1]))
            elif p == 1:  # вверх (гэп в seq2)
                for a1, a2 in backtrack(i-1, j):
                    results.append((a1 + seq1[i-1], a2 + '-'))
            elif p == 2:  # влево (гэп в seq1)
                for a1, a2 in backtrack(i, j-1):
                    results.append((a1 + '-', a2 + seq2[j-1]))
        return results

    alignments = backtrack(n, m)
    return alignments, score[n][m]

# Тест
v = "TCGATTA"
w = "TCATTG"
alns, scr = needleman_wunsch_all(v, w)
print(f"Оптимальный скор: {scr}")
for a1, a2 in alns:
    print(a1)
    print(a2)
    print()
```

Убедитесь, что найдено одно или несколько выравниваний.

## Часть 4. Алгоритм Смита–Ватермана (локальное выравнивание) — ручной разбор

### 4.1. Пример из лекции

Возьмём последовательности с общим участком:

```

w = "TGGTCGTG"
```

Попробуйте найти общий участок вручную.

### 4.2. Ручное заполнение таблицы с учётом нуля

В алгоритме Смита–Ватермана мы разрешаем обнулять отрицательные значения. Это позволяет "перезапускать" выравнивание.

**Правила:**
- Первая строка и первый столбец заполняются нулями
- Для каждой ячейки `(i,j)`:

```
diag = score[i-1][j-1] + (match если совпадают, иначе mismatch)
up = score[i-1][j] + gap
left = score[i][j-1] + gap
score[i][j] = max(0, diag, up, left)
```

- Максимальный скор ищется во всей таблице, не обязательно в правом нижнем углу
- Backtracking начинается с максимума и идёт до первого нуля

### 4.3. Реализация на Python

```python
def smith_waterman_all(seq1, seq2, match=1, mismatch=-1, gap=-1):
    n, m = len(seq1), len(seq2)
    score = np.zeros((n+1, m+1), dtype=int)
    pointers = [[[] for _ in range(m+1)] for _ in range(n+1)]
    max_score = 0
    max_positions = []

    for i in range(1, n+1):
        for j in range(1, m+1):
            diag = score[i-1][j-1] + (match if seq1[i-1] == seq2[j-1] else mismatch)
            up = score[i-1][j] + gap
            left = score[i][j-1] + gap
            max_val = max(0, diag, up, left)
            score[i][j] = max_val
            if max_val > 0:
                if diag == max_val:
                    pointers[i][j].append(0)
                if up == max_val:
                    pointers[i][j].append(1)
                if left == max_val:
                    pointers[i][j].append(2)
            # Отслеживаем максимумы
            if max_val > max_score:
                max_score = max_val
                max_positions = [(i, j)]
            elif max_val == max_score and max_val > 0:
                max_positions.append((i, j))

    # Рекурсивный сбор выравниваний от каждой максимальной позиции до нуля
    def backtrack(i, j):
        if score[i][j] == 0:
            return [("", "")]
        results = []
        for p in pointers[i][j]:
            if p == 0:
                for a1, a2 in backtrack(i-1, j-1):
                    results.append((a1 + seq1[i-1], a2 + seq2[j-1]))
            elif p == 1:
                for a1, a2 in backtrack(i-1, j):
                    results.append((a1 + seq1[i-1], a2 + '-'))
            elif p == 2:
                for a1, a2 in backtrack(i, j-1):
                    results.append((a1 + '-', a2 + seq2[j-1]))
        return results

    all_alignments = []
    for i, j in max_positions:
        all_alignments.extend(backtrack(i, j))
    return all_alignments, max_score

# Тест
v = "TGCTCGTA"
w = "TGGTCGTG"
alns, scr = smith_waterman_all(v, w)
print(f"Максимальный локальный скор: {scr}")
for a1, a2 in alns:
    print(a1)
    print(a2)
    print()
```

## Часть 5. Аффинный штраф за разрыв

### 5.1. Концепция

**Линейный штраф:** каждому гэпу одинаковый вес.

**Аффинный:** открытие гэпа (gap open) стоит дорого, продление (gap extend) — дёшево. Это биологически обосновано: одна мутация может вставить сразу несколько нуклеотидов, поэтому длинные гэпы вероятнее, чем много коротких.

### 5.2. Реализация с тремя матрицами

Полная реализация аффинного штрафа требует трёх матриц (средняя, верхняя, нижняя). Это сложно, но в Biopython это уже реализовано. Мы просто будем использовать готовые функции.

### 5.3. Визуализация влияния штрафов

Создадим функцию, которая для разных параметров строит график зависимости длины гэпов от штрафа (упрощённо).

```python
def test_gap_penalties(seq1, seq2, gap_open_list, gap_extend, matrix="BLOSUM62"):
    blosum = substitution_matrices.load(matrix)
    scores = []
    for go in gap_open_list:
        alns = pairwise2.align.globalds(seq1, seq2, blosum, go, gap_extend)
        # Берём первое выравнивание
        a1, a2, score, begin, end = alns[0]
        # Считаем количество гэпов
        gaps = a2.count('-') + a1.count('-')
        scores.append((go, score, gaps))
    return scores

# Тестовые последовательности (гемоглобин человека и мыши)
seq1 = str(human_seq)
seq2 = str(mouse_seq)

gap_opens = [-5, -10, -15, -20]
results = test_gap_penalties(seq1, seq2, gap_opens, -1)

for go, score, gaps in results:
    print(f"Gap open = {go:3d}: score = {score:4d}, gaps = {gaps:2d}")

# График
plt.plot(gap_opens, [r[1] for r in results], marker='o', label='Score')
plt.xlabel("Gap open penalty")
plt.ylabel("Alignment score")
plt.title("Влияние штрафа за открытие гэпа")
plt.gca().invert_xaxis()  # так как штраф отрицательный
plt.show()
```

### 6.3. Ручной расчёт простой матрицы (как в лекции)

Воспроизведём пример из лекции с алфавитом `{A, B, C}` и блоком из 6 последовательностей. Это упражнение на понимание логики BLOSUM.

Допустим, у нас есть множественное выравнивание блока (как на слайде):

```
S1: A B A B
S2: B B B C
S3: B B C C
S4: B B A B
S5: B B C C
S6: B B A C
```

Посчитаем наблюдаемые пары (без учёта порядка, т.е. AB и BA считаем одинаковыми):

- AA: 3
- AB: 8
- AC: 6
- BB: 26
- BC: 10
- CC: 7

Всего пар = 60.

**Частоты аминокислот:**
- A: 4
- B: 14
- C: 6

Всего символов = 24.

Ожидаемая частота пары AB (если независимы) = 2 * (fA * fB) = 2 * (4/24 * 14/24) = 2 * (56/576) = 112/576.
Наблюдаемая частота AB = 8/60.

Логарифмическое отношение правдоподобия (log-odds) в битах (основание 2):
```
2 * log2( (8/60) / (112/576) ) = 2 * log2( (8/60) * (576/112) ) = 2 * log2( (8*576)/(60*112) )
```

Вычислим в Python:

```python
import math
obs_ab = 8/60
exp_ab = (4/24)*(14/24)*2  # *2 потому что пары неупорядоченные
lod = 2 * math.log2(obs_ab / exp_ab)
print(f"Log-odds для AB: {lod:.2f}")
# Должно получиться около -1.09 (как в лекции)
```

### 6.4. Почему триптофан и цистеин имеют высокие значения на диагонали?

В матрице BLOSUM62 значения на диагонали (замена аминокислоты на саму себя) для W (триптофан) = 11, для C (цистеин) = 9. Это выше, чем для других.

- **W:** редкая аминокислота, большая, ароматическая. Любая замена с большой вероятностью нарушит структуру белка, поэтому естественный отбор сохраняет её.
- **C:** образует дисульфидные связи. Замена цистеина разрушает эти ковалентные связи, что критично для структуры.

Проверим в Python:

```python
bl62 = substitution_matrices.load("BLOSUM62")
print("W->W:", bl62["W"]["W"])
print("C->C:", bl62["C"]["C"])
print("A->A:", bl62["A"]["A"])  # для сравнения
```

## Часть 7. Использование Bio.pairwise2 для разных сценариев

### 7.1. Функции pairwise2

В Biopython есть несколько функций:

| Функция | Описание |
|---------|----------|
| `globalxx` | match=1, mismatch=0, gap=0 (просто идентичность) |
| `globalmx` | match=1, mismatch=0, gap с штрафом |
| `globalms` | match, mismatch, open/extend gap (линейный, если open=extend) |
| `globalds` | с матрицей замен |

Аналогично для локального: `localxx`, `localms`, `localds`.

### 7.2. Пример: сравнение двух близких последовательностей разными методами

```python
s1 = "HELLO WORLD"
s2 = "HELLO WORLd"  # маленькая буква d

# globalxx (только совпадения)
aln = pairwise2.align.globalxx(s1, s2)[0]
print(format_alignment(*aln))

# globalms (с штрафами)
aln = pairwise2.align.globalms(s1, s2, 2, -1, -1, -1)[0]
print(format_alignment(*aln))
```

### 7.3. Все возможные выравнивания с одинаковым скором

`pairwise2.align.globalms` возвращает список всех оптимальных выравниваний. Посмотрим, сколько их:

```python
alns = pairwise2.align.globalms("ATGC", "ATG", 1, -1, -1, -1)
```

## Часть 8. Транзиции и трансверсии (модель Кимуры)

### 8.1. Концепция

В нуклеотидных последовательностях замены делятся на:
- **Транзиции** (пурин↔пурин, пиримидин↔пиримидин)
- **Трансверсии** (пурин↔пиримидин)

Транзиции происходят чаще, поэтому в некоторых моделях (например, Кимуры) им дают разный вес.

### 8.2. Реализация скоринга с учётом транзиций/трансверсий

Напишем функцию, которая для заданной пары нуклеотидов возвращает скор с учётом типа замены.

```python
def kimura_score(a, b, match=1, transi=0.5, transv=-1):
    if a == b:
        return match
    purines = {'A', 'G'}
    pyrimidines = {'C', 'T'}
    if (a in purines and b in purines) or (a in pyrimidines and b in pyrimidines):
        return transi  # транзиция
    else:
        return transv  # трансверсия

# Пример
print(kimura_score('A', 'G', transi=0, transv=-2))  # 0
print(kimura_score('A', 'C', transi=0, transv=-2))  # -2
```
