## Введение

Сборка генома de novo (от лат. "заново") — это процесс восстановления полной последовательности ДНК организма из множества коротких фрагментов, полученных в результате секвенирования, без использования референсного генома. Это одна из самых сложных задач в биоинформатике, сравнимая со сборкой гигантского пазла, у которого нет ни картинки, ни рамки, и неизвестно даже точное количество деталей.

## Часть 1: Зачем нужно собирать геномы?

### Фундаментальная наука

**Поиск новых генов и белков:** Собранный геном позволяет идентифицировать ранее неизвестные гены, которые могут кодировать белки с важными функциями. Это расширяет наше понимание того, как устроены живые организмы.

**Систематика и филогенетика (изучение эволюции):** Сравнивая геномы разных видов, ученые могут строить эволюционные деревья и понимать, как организмы связаны друг с другом, когда они разошлись от общего предка и какие генетические изменения привели к появлению новых видов.

### Медицина и изучение человека

**Изучение внутривидового разнообразия:** Когда у нас есть один качественно собранный референсный геном (эталонная последовательность), мы можем сравнивать с ним геномы других особей того же вида.

**Поиск мутаций:** Сравнение с референсом помогает находить патологические мутации — изменения в ДНК, которые вызывают наследственные заболевания или повышают предрасположенность к ним.

**Генетическое тестирование:** Все коммерческие тесты, которые позволяют узнать о происхождении, рисках заболеваний или непереносимости лекарств, основаны на сравнении ДНК клиента с референсным геномом.

**Пренатальный скрининг:** Анализ геномов будущих родителей позволяет оценить риски рождения ребенка с наследственными заболеваниями.

**Персонализированная медицина (онкология):** Ключевое применение — лечение рака. У каждой опухоли есть уникальный набор соматических мутаций (приобретенных, а не унаследованных). Сравнивая геном опухолевой клетки с геномом здоровой клетки того же пациента, можно определить эти мутации и подобрать таргетную терапию, которая будет эффективна против конкретного типа рака.

## Часть 2: Основные понятия и терминология

**Рид (Read):** Это "сырые" данные, которые выдает секвенатор — короткая последовательность нуклеотидов, прочитанная с одного фрагмента ДНК. Длина рида зависит от технологии секвенирования (от 50–150 букв у Illumina до десятков тысяч у Oxford Nanopore).

**Контиг (Contig, от contiguous — примыкающий):** Результат сборки нескольких перекрывающихся ридов в один более длинный, непрерывный участок последовательности. Это как если бы вы собрали несколько соседних кусочков пазла в один блок.

**Скаффолд (Scaffold, или каркас):** Следующий уровень организации. Это несколько контигов, которые, как мы знаем из дополнительных данных, должны располагаться рядом друг с другом на хромосоме, но точная последовательность ДНК в промежутках между ними неизвестна. Эти неизвестные участки обозначаются буквами **N**.

**Картирование (Mapping):** Процесс размещения скаффолдов на физической карте генома. Такая карта может быть получена независимыми методами (генетическими или цитогенетическими), что позволяет привязать последовательности к конкретным местам на хромосомах.

**Покрытие (Coverage):** Показывает, сколько раз в среднем прочитан каждый нуклеотид генома. Рассчитывается как:

```
Покрытие = (длина рида × количество ридов) / длина генома
```

Высокое покрытие (например, 30x, 50x) необходимо для надежного определения консенсусной последовательности и исправления ошибок секвенирования.

## Часть 3: Проект «Геном человека»

Это самый амбициозный биологический проект в истории, который стал отправной точкой для всей современной геномики.

### Исторический контекст и сроки

| Параметр | Значение |
|---|---|
| Задуман | Начало 1980-х годов |
| Основной метод | Секвенирование по Сэнгеру |
| Бюджет | ~3 миллиарда долларов США |
| Черновая версия | Февраль 2001 года |
| Официальное завершение | Апрель 2003 года |
| Финальная публикация | 2004 год |

**Участники:** Крупнейшая международная коллаборация: США (NIH, Министерство энергетики), Великобритания (Институт Сангера), Япония, Франция, Германия, Китай.

**Директор на старте:** Джеймс Уотсон (один из открывателей структуры ДНК), который позже был отстранен от руководства.

### Чей геном секвенировали? Мифы и реальность

В Нью-Йорке было дано объявление о поиске волонтеров. Критерием отбора было смешанное происхождение (*blended ancestry*). Около **70% последовательности** было получено от одного человека — мужчины африкано-европейского происхождения, известного под кодовым именем **RP11**.

### Методология: BAC-to-BAC

1. **Создание библиотек фрагментов:** Геномную ДНК разрезали на крупные фрагменты (десятки тысяч пар оснований).
2. **Клонирование в векторы:** Фрагменты вставляли в бактериальные искусственные хромосомы (**BACs**). *E. coli* стала "фабрикой" по производству копий.
3. **Физическое картирование (фингерпринтинг):** Каждый BAC-клон обрабатывали рестриктазами. Полученные фрагменты разделяли электрофорезом, получая уникальный "фингерпринт". Совпадение фингерпринтов двух клонов указывало на перекрывающиеся участки ДНК.
4. **Построение карты:** Путем сравнения тысяч фингерпринтов была построена физическая карта, покрывающая все хромосомы.
5. **Массовое секвенирование:** Каждый BAC-клон фрагментировали и секвенировали методом Сэнгера на автоматических секвенаторах.

### Драма: Крейг Вентер и конкуренция с Celera Genomics

**Крейг Вентер** — блестящий ученый-предприниматель, основатель частной компании Celera Genomics — предложил метод **«шотган-секвенирования всего генома»** (*whole-genome shotgun sequencing*): просто порубить весь геном на мелкие кусочки и собрать с помощью мощных компьютеров, без физического картирования.

**Конфликт:** Вентер планировал патентовать гены и продавать доступ к данным, что противоречило принципу государственного проекта — все данные в открытом доступе.

**Результат:** В 2001 году оба проекта одновременно опубликовали черновики. Без давления Вентера публикация состоялась бы на несколько лет позже. Его метод хорошо работает для малых геномов, но для огромного, полного повторов человеческого генома без предварительного картирования работал плохо.

### Что мы узнали из генома человека?

| Параметр | Значение |
|---|---|
| Размер | 3.1 миллиарда пар оснований |
| Белок-кодирующие гены | ~20–25 тысяч |
| Экзоны | 1.5% генома |
| Интроны | ~24% генома |
| Повторы (транспозоны, LINE, SINE) | ~50% генома |
| Регуляторные области | ~5% генома |

> **Неожиданное открытие:** Раньше предполагали ~100 000 генов. Оказалось лишь ~20–25 тысяч. Сложность организма определяется не количеством генов, а сложностью их регуляции.

**Псевдогены:** У человека ~800 генов обонятельных рецепторов, но работают только 300. У слона — около 1200.

**Сравнение с мышью:** За ~75 миллионов лет эволюции куски хромосом сильно перемешались. X-хромосома оказалась наиболее консервативной.

**Геномные браузеры:**
- [UCSC Genome Browser](https://genome.ucsc.edu/)
- [Ensembl](https://www.ensembl.org/)

## Часть 4: Математические основы сборки (Алгоритмы и графы)

### 4.1. Подход 1: Граф перекрытий (Overlap Graph) — для длинных ридов

- **Принцип:** Каждый рид — вершина графа. Если два рида перекрываются концами, между ними рисуется ребро.
- **Задача:** Найти **Гамильтонов путь** — путь через каждую вершину ровно один раз.
- **Применимость:** Длинные риды (Sanger, PacBio, Oxford Nanopore).
- **Недостаток:** Задача поиска Гамильтонова пути является **NP-полной** — вычислительно очень сложной.

### 4.2. Подход 2: Граф де Брёйна (De Bruijn Graph) — для коротких ридов

- **Идея:** Все риды нарезаются на **k-меры** (подпоследовательности фиксированной длины k).
- **Пример:** Рид `ATGGC` при k=3 → k-меры: `ATG`, `TGG`, `GGC`.

**Построение графа:**
- **Вершины** = уникальные k-меры
- **Ребра** = связи между последовательными k-мерами в исходных ридах

- **Задача:** Найти **Эйлеров путь** — путь по каждому ребру ровно один раз (эффективно решается, не NP-полная).

**Преимущества:**
- Компактный граф
- Эффективное алгоритмическое решение

**Недостатки:**
- Выбор длины **k** критически важен (слишком мало → неразличимые повторы; слишком много → граф распадается)
- Плохо справляется с повторами
- Гетерозиготность создает "пузырьки" (bubbles) в графе

## Часть 5: Современные технологии для улучшения сборки

### 5.1. Мате-пэйрное секвенирование (Mate-Pair Sequencing)

Метод для соединения контигов в скаффолды через небольшие промежутки и повторы.

1. Получают длинные фрагменты ДНК (2–5 тыс. пар оснований).
2. Концы фрагментов метят **биотином**.
3. Фрагменты циркуляризируют (замыкают в кольцо).
4. Кольцевую ДНК фрагментируют на короткие кусочки.
5. С помощью **стрептавидина** вытягивают только биотинилированные концы.
6. Секвенируют. Два рида из одной пары → два контига находятся на известном расстоянии.

### 5.2. Hi-C (Захват конформации хроматина)

Позволяет понять трёхмерную организацию хромосом в ядре клетки.

1. Клетки обрабатывают **формальдегидом** — он "сшивает" пространственно близкие участки ДНК.
2. ДНК извлекают, режут и лигируют.
3. Образуются **химерные молекулы** из пространственно близких, но линейно далеких участков.
4. Анализ всех пар ридов строит **матрицу контактов**, показывающую:
   - Какие контиги принадлежат одной хромосоме
   - Порядок контигов на хромосоме
   - Границы между хромосомами

### 5.3. Секвенирование одной молекулы (Linked-Reads, например, 10x Genomics)

1. Длинные молекулы ДНК распределяют по **капелькам эмульсии** (одна молекула — одна капелька).
2. В каждую капельку помещают шарик с уникальным **баркодом**.
3. Внутри капельки молекулу фрагментируют и амплифицируют — все фрагменты получают один баркод.
4. Секвенируют на Illumina.
5. По баркодам известно, какие риды происходят из одной длинной молекулы.

**Применение:** Разделение отцовского и материнского гаплотипов, выявление структурных вариантов (делеций).

### 5.4. Прорыв: Сборка «От теломеры до теломеры» (T2T — Telomere-to-Telomere)

**Ключевые технологии:**
- **PacBio HiFi:** точные прочтения длиной 10–20 тыс. оснований
- **Oxford Nanopore:** прочтения в сотни тысяч и миллионы оснований

**Алгоритм:** Возврат к графу перекрытий (Overlap Graph) и Гамильтонову пути — ридов немного, но они очень длинные и покрывают повторяющиеся регионы целиком.

**Результат — геном T2T-CHM13:**
- Все 24 хромосомы представлены **24 скаффолдами** (каждая хромосома = один непрерывный кусок)
- Впервые полностью прочитаны центромеры и теломеры

> [Проект T2T на GitHub](https://github.com/marbl/CHM13)

## Часть 7: Контроль качества сборки (Метрики)

### 7.1. Покрытие (Coverage)

- **Среднее покрытие** = (общая длина ридов) / (длина генома)
- Каждая позиция должна быть прочитана многократно (>10x) для надёжного определения нуклеотида
- Участки с высоким GC-составом часто имеют пониженное покрытие

### 7.2. Метрики непрерывности: N50, NG50, L50

**N50 — алгоритм:**
1. Взять все контиги, отсортировать по убыванию длины
2. Складывать длины, начиная с самого длинного
3. Остановиться, когда накопленная сумма достигнет **50% общей длины сборки**
4. Длина последнего добавленного контига = **N50**

> **Смысл:** Половина генома находится в контигах длиной ≥ N50. Чем выше N50 — тем лучше.

**NG50:** Аналог N50, но за 100% берётся известный размер генома (честнее при неполной сборке).

**L50:** Количество контигов, составляющих эти 50%. Идеальное значение: **L50 = 1**.

*Пример:* N50 = 5 млн, L50 = 3 → половина генома собрана в трёх самых длинных контигах.

### 7.3. Полнота (Completeness)

- **Процент от ожидаемого размера:** длина сборки / оценочный размер генома
- **BUSCO** — стандартный инструмент: проверяет наличие ультраконсервативных генов, которые должны быть у всех представителей группы. Результат ≥95% = очень полная сборка. [busco.ezlab.org](https://busco.ezlab.org/)

### 7.4. Корректность (Correctness)

Исходные риды выравнивают обратно на собранный геном и ищут противоречия — **химеры** (ошибочно соединённые фрагменты из разных частей генома). Инструменты: **REAPR**, **ValidateAssembly**.

## Часть 8: Современные программы-сборщики

### SPAdes (СПАдес)
- **Для чего:** Сборка геномов бактерий, вирусов, небольших эукариот
- **Вход:** Короткие риды Illumina + гибридная сборка с длинными ридами
- **Разработка:** Лаборатория алгоритмической биологии, Санкт-Петербург
- **Авторы:** Сергей Нурк, Павел Певзнер, Николай Вяхи, Алексей Гуревич и др.
- [GitHub SPAdes](https://github.com/ablab/spades)

### Verkko (Веркко)
- **Для чего:** Сборка крупных геномов T2T с длинными ридами (PacBio HiFi + Oxford Nanopore)
- **Название:** "Сеть" по-фински
- **Разработка:** Совместный проект, включая Сергея Нурка и Сергея Корина

## Заключение и итоги

Сборка генома de novo — это сложнейшая вычислительная и экспериментальная задача. Идеальная сборка практически недостижима для больших геномов с повторами, но современные методы позволяют к ней приближаться.

- **Короткие риды (Illumina) + графы де Брёйна** — основа массового секвенирования, но не справляются с повторами
- **Длинные риды (PacBio, ONT) + графы перекрытий** — ключ к решению проблемы повторов
- **Вспомогательные методы (Mate-Pair, Hi-C, Linked-Reads)** — информация о структуре на больших расстояниях
- **Проект «Геном человека»** — заложил основы всей современной биологии и персонализированной медицины
- **Контроль качества (N50, BUSCO)** — неотъемлемая часть любой сборки
- **Будущее — пангеномы:** множество геномов, представляющих всё разнообразие популяции

## Глоссарий терминов

| Термин | Определение |
|---|---|
| **BAC** | Бактериальная искусственная хромосома. Вектор для клонирования крупных фрагментов ДНК (до 300 тыс. п.н.) в клетках *E. coli* |
| **BUSCO** | Программа для оценки полноты сборки через поиск ультраконсервативных генов |
| **Celera Genomics** | Частная компания Крейга Вентера, секвенировавшая геном человека методом шотган |
| **Contig (Контиг)** | Непрерывная последовательность ДНК из перекрывающихся ридов. Без неизвестных участков |
| **Coverage (Покрытие)** | Среднее число прочтений каждого нуклеотида генома |
| **De Bruijn Graph** | Граф, где вершины — k-меры, рёбра — их перекрытия. Основа сборки коротких ридов |
| **De novo assembly** | Сборка генома без использования референсной последовательности |
| **Eulerian Path** | Путь по каждому ребру графа ровно один раз. Лежит в основе графа де Брёйна |
| **Fingerprint** | Уникальный паттерн фрагментов ДНК после обработки рестриктазами и электрофореза |
| **GC-content** | Процент гуанина (G) и цитозина (C) в последовательности ДНК |
| **Hamiltonian Path** | Путь через каждую вершину графа ровно один раз. Основа графа перекрытий |
| **Hi-C** | Метод захвата конформации хроматина для изучения 3D-структуры хромосом |
| **k-mer** | Подпоследовательность ДНК фиксированной длины k |
| **L50** | Количество самых длинных контигов, составляющих 50% длины сборки |
| **Linked-Reads** | Короткие риды, помеченные баркодами принадлежности к одной длинной молекуле (10x Genomics) |
| **Long-read sequencing** | Секвенирование с длинными прочтениями (PacBio, Oxford Nanopore) |
| **Mate-Pair** | Метод секвенирования концов длинных фрагментов для соединения контигов в скаффолды |
| **N50** | Длина контига, на котором накопленная сумма длин достигает 50% всей сборки |
| **NG50** | Аналог N50, но за 100% берётся размер генома, а не длина сборки |
| **NCBI** | Национальный центр биотехнологической информации США. [ncbi.nlm.nih.gov](https://www.ncbi.nlm.nih.gov/) |
| **Overlap Graph** | Граф, где вершины — риды, рёбра — их перекрытия. Для длинных ридов |
| **PacBio** | Технология секвенирования одной молекулы (SMRT), длинные риды до 20–30 тыс. п.н. |
| **PCR** | Полимеразная цепная реакция. Метод многократного копирования участка ДНК |
| **Pseudogene (Псевдоген)** | Копия гена, утратившая функцию из-за накопленных мутаций |
| **QUAST** | Программа для расчёта метрик качества сборки геномов |
| **Read (Рид)** | Последовательность нуклеотидов, полученная за одно прочтение фрагмента ДНК |
| **Reference genome** | Эталонная последовательность генома для сравнения и аннотации |
| **Restriction enzyme (Рестриктаза)** | Фермент, разрезающий ДНК в строго определённых сайтах |
| **Sanger sequencing** | Метод секвенирования на основе дидезоксинуклеотидов. Первое поколение |
| **Scaffold (Скаффолд)** | Несколько контигов известного порядка с промежутками, заполненными N |
| **SNP** | Однонуклеотидный полиморфизм. Замена одного нуклеотида, встречающаяся >1% в популяции |
| **SPAdes** | Популярный сборщик геномов для бактерий, разработан в Санкт-Петербурге |
| **Streptavidin** | Белок с исключительно высоким сродством к биотину. Используется для очистки |
| **T2T** | Telomere-to-Telomere. Сборка, где каждая хромосома — одна непрерывная последовательность |
| **TADs** | Топологически ассоциированные домены. Структурные единицы организации хроматина |
| **Telomere (Теломера)** | Концевой участок хромосомы из повторяющихся последовательностей |
| **Transposon (Транспозон)** | "Прыгающий ген". Последовательность ДНК, способная перемещаться в геноме |
| **UCSC Genome Browser** | Геномный браузер Калифорнийского университета. [genome.ucsc.edu](https://genome.ucsc.edu/) |
| **Verkko** | Сборщик геномов для T2T с длинными ридами |
| **Whole-genome shotgun** | Стратегия случайной фрагментации всего генома с последующей компьютерной сборкой |
| **YAC** | Дрожжевая искусственная хромосома. Вектор для клонирования очень крупных фрагментов |
